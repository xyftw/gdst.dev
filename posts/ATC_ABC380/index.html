🔗 A - 123233 (abc380 A)

tags: 計數(Counting)

題意

給定一個 666 位數的正整數 NNN，判斷 NNN 是否滿足下列所有條件：

  • 在 NNN 的數字中，數字 111 恰好出現 111 次。
  • 在 NNN 的數字中，數字 222 恰好出現 222 次。
  • 在 NNN 的數字中，數字 333 恰好出現 333 次。

約束條件

  • NNN 為一個整數，滿足 100000≤N≤999999100000 \le N \le 999999100000≤N≤999999。

思路：計數(Counting)

將 NNN 轉換為字串，並統計每個數字出現的次數，最後判斷是否滿足條件即可。

複雜度分析

  • 時間複雜度：O(6)O(6)O(6)。
  • 空間複雜度：O(10)O(10)O(10)。

🔗 B - Hurdle Parsing (abc380 B)

tags: 分組循環

題意

給定一個字串 SSS，其中僅包含 | 和 -，分別輸出每個 | 之間的 - 的長度。

約束條件

  • SSS 是一個長度在 333 到 100100100 之間（包含 333 和 100100100）的字串。
  • SSS 中至少包含一個 -。

思路：分組循環

我們要計算每段連續的 - 的長度，因此可以透過 分組循環 來處理。

  • 維護一個指標 iii，從 111 開始遍歷字串。
  • 紀錄每一段 - 的起始位置 ststst，並向後尋找直到遇到 | 或到達字串的結尾。
  • 計算該段 - 的長度，並將其加入答案陣列。
  • 將指標 iii 往後移動。

此外，由於 SSS 一定是由 | 開始，因此可以從索引 111 開始處理，否則需要跳過第一段的長度。

複雜度分析

  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 是字串的長度。
  • 空間複雜度：O(k)O(k)O(k)，其中 kkk 是字串中 | 的數量減 111，也就是答案陣列的長度。在最壞情況下，kkk 可能接近 n2\frac{n}{2}2n​。

🔗 C - Move Segment (abc380 C)

tags: 字串(String) 分組循環

題意

給定一個長度為 NNN 的二進位制字串 SSS，其由 0 和 1 組成。

將連續的 1 視為一個 blocks，將字串 SSS 中從開頭數來的第 KKK 個 111-block 移動到緊接著第 (K−1)(K-1)(K−1) 個 111-block 之後，並輸出結果字串。

約束條件

  • 1≤N≤5×1051 \leq N \leq 5 \times 10^51≤N≤5×105
  • 2≤K2 \leq K2≤K，且 SSS 至少包含 KKK 個 111-blocks。

思路：分組循環

這題和前一題類似，需要找到每一個連續的 1 組成的 blocks，並將第 KKK 個 block 移動到第 (K−1)(K-1)(K−1) 個 block 之後。

因此可以透過 分組循環 來找到每一個 block 的起始和結束位置 [l,r][l, r][l,r]，並將第 KKK 個 block 移動到第 (K−1)(K-1)(K−1) 個 block 之後。

令第 KKK 個 block 的起始和結束位置為 [l,r][l, r][l,r]，第 (K−1)(K-1)(K−1) 個 block 的結束位置為 rprevr_{prev}rprev​，則需要交換的區間分別為 [l,r][l, r][l,r] 和 [rprev+1,l−1][r_{prev} + 1, l - 1][rprev​+1,l−1]，直接交換即可。

複雜度分析

  • 時間複雜度：O(N)O(N)O(N)，其中 NNN 是字串的長度。
  • 空間複雜度：O(N)O(N)O(N)。

🔗 D - Strange Mirroring (abc380 D)

tags: 迭代(Iteration) 遞迴(Recursion)

題意

給定一個由大寫和小寫英文字母組成的字串 SSS，對 SSS 執行以下操作 1010010^{100}10100 次：

  • 構建一個字串 TTT，其內容為 SSS 中的大寫字母變成小寫，小寫字母變成大寫後的結果。
  • 將 TTT 附加到 SSS 後面。

回答 QQQ 個查詢。第 iii 個查詢如下：

  • 找出所有操作完成後，從 SSS 開始的第 KiK_iKi​ 個字符。

約束條件

  • ∣S∣≤2×105|S| \leq 2 \times 10^5∣S∣≤2×105
  • 1≤Q≤2×1051 \le Q \le 2 \times 10^51≤Q≤2×105
  • 1≤Ki≤10181 \le K_i \le 10^{18}1≤Ki​≤1018

思路

由於操作次數達到 1010010^{100}10100，直接模擬這麼大次數的操作顯然不可行，因此需要找到一種更高效的方式來確定查詢位置的字符。

首先，觀察每次操作的規則：將目前的字串 SSS 中的大寫字母變成小寫，小寫字母變成大寫後，附加到原字串的末尾。這意味著每進行一次操作，字串的長度會翻倍。因此，在 mmm 次操作後，字串的長度將達到 n×2mn \times 2^mn×2m。而我們可以利用這個性質來確定查詢位置 KiK_iKi​ 所在的字串是位於第 mmm 次操作後的原始部分 SSS 或翻轉部分 TTT 。

舉例來說，令 0 表示未翻轉，1 表示翻轉。則對於字串 aB 而言，前 444 次操作如下：

  1. 00
  2. 0011
  3. 00111100
  4. 0011110011000011
    對於 K=15K = 15K=15 而言，他上次是由 K=7K = 7K=7 翻轉而來；再上次是 K=3K = 3K=3，再再上次是 K=1K = 1K=1，這已經在原始字串中，因此不再翻轉。故 K=15K = 15K=15 相對 K=1K = 1K=1 翻轉了 333 次。

因此，可以維護長度 [n,2n,4n,8n,16n,...][n, 2n, 4n, 8n, 16n, ...][n,2n,4n,8n,16n,...]，由後往前檢查 KiK_iKi​ 所在的位置是否屬於原始字串 SSS 或是其翻轉後的部分 TTT。如果 KiK_iKi​ 落在翻轉後的部分 TTT，我們將其映射回對應的原始位置，並記錄翻轉的次數 cntcntcnt，直到找到原始字串中的字符為止。

最終，根據翻轉的總次數 cntcntcnt 的奇偶性來決定是否需要對該字符進行大小寫反轉即可。

複雜度分析

  • 時間複雜度：O(Qlog⁡M)O(Q \log M)O(QlogM)，其中 QQQ 是查詢的數量，M=max⁡(K)M = \max(K)M=max(K) 是查詢位置的最大值。
  • 空間複雜度：O(log⁡M)O(\log M)O(logM)。

🔗 E - 1D Bucket Tool (abc380 E)

tags: 併查集(Disjoint Set)

題意

從左到右有 NNN 個兩兩相鄰且連續排列的格子，編號為 111 到 NNN。最初，第 iii 個格子的顏色為 iii。

處理 QQQ 個查詢：

  • 1 x c: 將 xxx 號格子及其相鄰的、顏色和 xxx 號格子相同的格子塗上顏色 ccc。
  • 2 c: 輸出被塗上顏色 ccc 的單元格數量。

約束條件

  • 1≤N≤5×1051 \leq N \leq 5 \times 10^51≤N≤5×105
  • 1≤Q≤2×1051 \leq Q \leq 2 \times 10^51≤Q≤2×105

思路：區間併查集(Disjoint Set)

我們可以將顏色相同的且相鄰的格子視為一個連續區間，並且用 併查集(Disjoint Set) 來維護這些區間。除了維護每個節點所屬的區間編號 papapa 、以及每個區間的大小 szszsz 外，由於需要處理操作 2，還需要額外維護每個區間的顏色 colorcolorcolor 以及每個顏色的格子數量 cntcntcnt。

由於在操作 1 中，我們需要將某個格子及其相鄰的、顏色和該格子相同的格子塗上顏色 ccc，而在對該區間塗色後，又可能需要和相鄰的區間合併，因此需要檢查是否需要合併左側和右側區間。為此，我們可以在 union 操作時改成依照每個區間的左端點合併，即該區間的左端點為 fx=find(x)f_x = \text{find}(x)fx​=find(x)，由於維護的是一個連續區間，因此也可以得右端點為 fx+sz[fx]−1f_x + sz[f_x] - 1fx​+sz[fx​]−1。故合併時只需要檢查 left=fx−1left = f_x - 1left=fx​−1 和 right=fx+sz[fx]right = f_x + sz[f_x]right=fx​+sz[fx​] 所屬的區間的顏色是否和新顏色 ccc 相同即可，如果相同則需要合併。

需要注意的是，合併後的區間左端點 fxf_xfx​ 可能會改變，因此需要先合併 rightrightright 再合併 leftleftleft，或是需要重新計算 fxf_xfx​ 的值。

具體步驟如下：

  • 對於操作 1 x c，我們首先找到格子 xxx 所在的區塊，檢查其顏色是否已經是 ccc。如果不是，則更新顏色，並調整 cntcntcnt 陣列：否則直接跳過即可。
    • 令原本的顏色為 cur_ccur\_ccur_c，更新後的顏色為 ccc。則 cnt[cur_c]cnt[cur\_c]cnt[cur_c] 減少了 xxx 所在區間的大小 sz[fx]sz[f_x]sz[fx​]，cnt[c]cnt[c]cnt[c] 增加了 xxx 所在區間的大小 sz[fx]sz[f_x]sz[fx​]。
    • 更新顏色後，我們檢查該區塊的左側和右側是否有相鄰且顏色相同的區塊，如果有，則將它們合併成一個更大的區塊。
  • 對於操作 2 c ，直接輸出 cnt[c]cnt[c]cnt[c] 即可。

複雜度分析

  • 時間複雜度：O(Q⋅α(N))O(Q \cdot \alpha(N))O(Q⋅α(N))
  • 空間複雜度：O(N)O(N)O(N)。

🔗 F - Exchange Game (abc380 F)

tags: 博弈論(Game Theory) 記憶化搜尋(Memoization) 狀態壓縮DP(Bitmask DP)

題意

有兩位玩家，他們將使用寫有數字的卡牌進行遊戲。最初，玩家 A 的手上有 NNN 張卡牌，數字分別為 A1,…,ANA_1, \ldots, A_NA1​,…,AN​；玩家 B 的手上有 MMM 張卡牌，數字分別為 B1,…,BMB_1, \ldots, B_MB1​,…,BM​；桌上有 LLL 張卡牌，數字分別為 C1,…,CLC_1, \ldots, C_LC1​,…,CL​。

在整個遊戲過程中，玩家 A 和玩家 B 都知道所有卡牌上的數字，包括對方手中的卡牌。

遊戲由玩家 A 先開始，他們輪流執行以下動作：

  • 從自己的手牌中選擇一張卡牌放到桌上。然後，如果桌上有卡牌的數字小於他剛放下的卡牌的數字，他可以從桌上取走其中一張這樣的卡牌回到自己的手中。

最先無法進行動作的玩家輸，另一個玩家贏。如果 雙方都以最優策略進行遊戲 ，判斷誰會獲勝。

約束條件

Constraints

  • 1≤N,M,L1 \leq N, M, L1≤N,M,L
  • N+M+L≤12N + M + L \leq 12N+M+L≤12

思路：博弈論(Game Theory) + 狀態壓縮DP(Bitmask DP) + 記憶化搜尋(Memoization)

由於卡牌的數量 N+M+L≤12N + M + L \leq 12N+M+L≤12，因此可以考慮狀態壓縮DP，維護兩個玩家的目前手牌以及輪到誰的回合三個狀態，其中手牌的狀態可以用一個整數的二進位表示。

在輪到某玩家時，可以枚舉可能的選擇，這包含兩個部分：

  1. 打出手中的一張牌
  2. 不回收任何牌、或是回收桌上比剛剛打出的牌小的牌。

由於雙方都以最優策略進行遊戲，因此如果任何一種選擇使得對方必輸，則自己必贏。

實現時，使用 記憶化搜尋(Memoization) 來實現即可。令 f(a,b,flag)f(a, b, flag)f(a,b,flag) 表示當前輪到 flagflagflag 玩家出牌，aaa 和 bbb 分別是兩個玩家的手牌，返回該玩家是否能獲勝。

具體步驟如下：

  1. 使用 Bitmask 來表示每張卡牌的所在位置：
    • 玩家 A 的手牌：用一個整數 aaa 的二進位表示，每一位代表一張卡牌是否在玩家 A 的手中。
    • 玩家 B 的手牌：用另一個整數 bbb 的二進位表示，每一位代表一張卡牌是否在玩家 B 的手中。
    • 桌面上的卡牌：通過 aaa 和 bbb 的補集  (a∣b)~(a | b) (a∣b) 來表示，即不在任何一位玩家手中的卡牌即在桌面上。不過由於有效的卡牌編號為 0,…,N+M+L−10, \ldots, N + M + L - 10,…,N+M+L−1，因此需要再對 mask=(1<<(N+M+L))−1mask = (1 << (N + M + L)) - 1mask=(1<<(N+M+L))−1 取交集。
  2. 在每一個遊戲狀態，需要考慮當前玩家可以執行的行動：
    1. 打出手上的牌：遍歷當前玩家手中的所有卡牌，選擇其中一張出到桌面上。若當前玩家無法出牌，則該玩家輸，對方獲勝。
    2. 不回收任何牌、或是回收桌上比剛剛打出的牌小的牌：
      • 不回收任何牌：僅將選中的牌放到桌面上，轉換到下一位玩家的回合。
      • 回收一張較小的牌：如果桌面上存在比剛出牌小的卡牌，可以選擇其中一張回收到自己的手中，然後轉換到下一位玩家的回合。
  3. 在這些行動中，如果存在一種行動使得對方必輸，則自己必贏；否則，若所有行動都使得對方必贏，則自己必輸。

複雜度分析

  • 時間複雜度：O((N+M+L)2⋅3N+M+L)O((N+M+L)^2 \cdot 3^{N+M+L})O((N+M+L)2⋅3N+M+L)
    • 每張卡牌有三種可能的狀態，因此狀態數量最多有 3N+M+L3^{N+M+L}3N+M+L 種
    • 每種狀態需要 O(N+M+L2)O({N+M+L}^2)O(N+M+L2) 的時間枚舉可能的選擇
  • 空間複雜度：O(3N+M+L)O(3^{N+M+L})O(3N+M+L)
