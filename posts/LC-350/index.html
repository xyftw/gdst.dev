🔗 🟢 350. Intersection of Two Arrays II

tags: 陣列(Array) 雜湊表(Hash Table) 排序(Sorting) 雙指針(Two Pointers) 二分搜尋(Binary Search)

題意

給定兩個整數陣列 nums1nums1nums1 和 nums2nums2nums2，以陣列形式返回它們交集的陣列。

返回結果中的每個元素出現的次數，應與元素在兩個陣列中都出現的次數一樣多。可以按照任意順序返回結果。

約束條件

  • 1≤nums1.length,nums2.length≤10001 \leq nums1.length, nums2.length \leq 10001≤nums1.length,nums2.length≤1000
  • 0≤nums1[i],nums2[i]≤10000 \leq nums1[i], nums2[i] \leq 10000≤nums1[i],nums2[i]≤1000

思路

方法一：基於值域或對索引取交集

由於每個數字出現的次數可能不只一次，我們需要記錄每個數字出現的次數。

注意到 nums1nums1nums1 和 nums2nums2nums2 中的數字範圍是 000 到 100010001000，我們可以使用這個特性來解決這個問題。

令 cnt1cnt1cnt1 和 cnt2cnt2cnt2 分別是長度為 100110011001 的計數陣列，用來記錄 nums1nums1nums1 和 nums2nums2nums2 中每個數字出現的次數。

則遍歷值域 000 到 100010001000，對於每個數字 xxx，其在交集陣列出現的次數為 min⁡(cnt1[x],cnt2[x])\min(cnt1[x], cnt2[x])min(cnt1[x],cnt2[x])。因此，我們只需要在答案陣列中添加 min⁡(cnt1[x],cnt2[x])\min(cnt1[x], cnt2[x])min(cnt1[x],cnt2[x]) 個 xxx 即可。

這個思路也能使用雜湊表來實現，對在 cnt1cnt1cnt1 和 cnt2cnt2cnt2 中的索引取交集，然後將交集中的數字按照次數添加到答案陣列中即可。

複雜度分析

  • 時間複雜度：O(n+m+D)\mathcal{O}(n + m + D)O(n+m+D)，其中 nnn 和 mmm 分別是 nums1nums1nums1 和 nums2nums2nums2 的長度，DDD 是值域大小，本題中為 100110011001。
  • 空間複雜度：O(D)\mathcal{O}(D)O(D)，需要額外的空間來存儲兩個計數陣列。

方法二：雜湊表(Hash Table)

方法一中我們先遍歷兩個陣列，然後再遍歷值域或索引的交集，但若 x∈nums1∧x∉nums2∨x∉nums1∧x∈nums2x \isin nums1 \land x \notin nums2 \lor x \notin nums1 \land x \isin nums2x∈nums1∧x∈/nums2∨x∈/nums1∧x∈nums2 ，則不需要考慮 xxx 。因此只需要遍歷兩個陣列各一次即可。

同樣先遍歷一次 nums1nums1nums1 ，令 cnt1cnt1cnt1 為 nums1nums1nums1 中每個數字出現的次數。在遍歷 nums2nums2nums2 的時候，對於每個數字 xxx ，若 cnt1[x]>0cnt1[x] > 0cnt1[x]>0 ，則將 xxx 加入答案陣列中，並將 cnt1[x]cnt1[x]cnt1[x] 減一。

計數的方式可以用 雜湊表(Hash Table) 來實現，但由於值域是 000 到 100010001000 ，也可以使用一個長度為 100110011001 的陣列來實現。

複雜度分析

  • 時間複雜度：O(n+m)\mathcal{O}(n + m)O(n+m)，其中 nnn 和 mmm 分別是 nums1nums1nums1 和 nums2nums2nums2 的長度。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) 或 O(D)\mathcal{O}(D)O(D) 。
    • 若使用雜湊表，空間複雜度是 O(n)O(n)O(n) 。
    • 若使用計數陣列，空間複雜度是 O(D)O(D)O(D) 。

方法三：排序(Sorting) + 雙指針(Two Pointers)

若 nums1nums1nums1 和 nums2nums2nums2 都是有序的，則可以使用 雙指針(Two Pointers) 的方式來解決這個問題。

首先對 nums1nums1nums1 和 nums2nums2nums2 進行排序，然後使用兩個指針 iii 和 jjj 分別指向 nums1nums1nums1 和 nums2nums2nums2 的開頭。

  • 若 nums1[i]<nums2[j]nums1[i] < nums2[j]nums1[i]<nums2[j] ，則 iii 往右移動。
  • 若 nums1[i]>nums2[j]nums1[i] > nums2[j]nums1[i]>nums2[j] ，則 jjj 往右移動。
  • 若 nums1[i]=nums2[j]nums1[i] = nums2[j]nums1[i]=nums2[j] ，則將 nums1[i]nums1[i]nums1[i] 加入答案陣列中，並將 iii 和 jjj 同時往右移動。

複雜度分析

  • 時間複雜度：O(nlog⁡n+mlog⁡m)\mathcal{O}(n \log n + m \log m)O(nlogn+mlogm)，其中 nnn 和 mmm 分別是 nums1nums1nums1 和 nums2nums2nums2 的長度。
    • 若已經是有序的，則時間複雜度是 O(n+m)\mathcal{O}(n + m)O(n+m)。
  • 空間複雜度：O(1)\mathcal{O}(1)O(1)，忽略排序的空間複雜度。
