🔗 🔴 3068. Find the Maximum Sum of Node Values 2268

tags: Biweekly Contest 125 位運算(Bit Manipulation) 動態規劃(DP) 樹形DP 狀態機DP

雖然方法一的 樹形DP 比較直覺，但在定義狀態跟轉移方程式時會比較複雜，看了好幾次才懂；知道結論後用方法二的 狀態機DP 會容易許多。本篇文章在敘述上可能有許多不完整之處，建議閱讀參考資料。

題意

有一棵包含 nnn 個節點的無向樹，其中節點編號從 000 到 n−1n - 1n−1。並給定一個長度為 n−1n - 1n−1 的二維整數陣列 edgesedgesedges，其中 edges[i]=[ui,vi]edges[i] = [u_i, v_i]edges[i]=[ui​,vi​] 表示樹中節點 uiu_iui​ 和 viv_ivi​ 之間有一條邊。同時給定一個正整數 kkk，以及一個長度為 nnn 的非負整數陣列 numsnumsnums，其中 nums[i]nums[i]nums[i] 表示節點 iii 的值。

你的目標是 最大化 樹中所有節點值之和。為了實現這一目標，你可以執行以下操作 任意 次（包括零次）：

  • 選擇任何一條連接節點 uuu 和 vvv 的邊 [u,v][u, v][u,v] ，並將 uuu 和 vvv 的值更新為：
    • nums[u]=nums[u]⊕knums[u] = nums[u] \oplus knums[u]=nums[u]⊕k
    • nums[v]=nums[v]⊕knums[v] = nums[v] \oplus knums[v]=nums[v]⊕k

返回通過執行以上操作 任意次 後，可以得到的 最大 可能總和。

思路

首先整理狀態，雖然每個點可以被操作任意次，但是由於 XOR 的性質 x⊕x=0x \oplus x = 0x⊕x=0 和 x⊕0=xx \oplus 0 = xx⊕0=x ，所以每個點的其實只有兩種狀態：操作偶數次 、操作奇數次。

此外，由於是無向樹，所以任何點都可以當作根節點，因此我們可以直接假定 000 為根節點。

方法一：樹形DP

因此，這題就變成了「選」或「不選」每條邊的動態規劃(DP)問題。我們可以用 樹形DP 的方式來解決這個問題。

  • 定義 f(u,0)f(u, 0)f(u,0) 和 f(u,1)f(u, 1)f(u,1) 分別表示在節點 uuu 操作偶數次和奇數次時，其子樹 (不包括 uuu) 的最大值。
    • 注意：這裡的 f(u,0)f(u, 0)f(u,0) 和 f(u,1)f(u, 1)f(u,1) 並 不等於 dfs(u,fa,0)dfs(u, fa, 0)dfs(u,fa,0) 和 dfs(u,fa,1)dfs(u, fa, 1)dfs(u,fa,1)，後者的 000 和 111 分別表示不操作 (u,fa)(u, fa)(u,fa) 和操作 (u,fa)(u, fa)(u,fa) 這條邊，所能得到的 uuu 的子樹(包含 uuu)的最大值。
  • 初始化 f(u,0)=0f(u, 0) = 0f(u,0)=0 和 f(u,1)=−∞f(u, 1) = -\inftyf(u,1)=−∞ ，因為一開始不考慮任何邊時， uuu 無法被操作，故用 −∞-\infty−∞ 來表示不合法的情況。

接著考慮 (u,fa)(u, fa)(u,fa) 這條邊是否被操作，所能得到的 uuu 的子樹 (包含 uuu) 的最大值，也就是 dfs(u,fa,0)dfs(u, fa, 0)dfs(u,fa,0) 和 dfs(u,fa,1)dfs(u, fa, 1)dfs(u,fa,1) 的值，000 和 111 分別表示不操作 (u,fa)(u, fa)(u,fa) 和操作 (u,fa)(u, fa)(u,fa) 這條邊，可以由 f(u,0)f(u, 0)f(u,0) 和 f(u,1)f(u, 1)f(u,1) 的值得到：

  • dfs(u,fa,0)=max⁡(f(u,0)+nums[u],f(u,1)+(nums[u]⊕k))dfs(u, fa, 0) = \max(f(u, 0) + nums[u], f(u, 1) + (nums[u] \oplus k))dfs(u,fa,0)=max(f(u,0)+nums[u],f(u,1)+(nums[u]⊕k))，表示不操作 (u,fa)(u, fa)(u,fa) 這條邊。
    • 由於 f(u,0)f(u, 0)f(u,0) 表示在 uuu 的子樹中， uuu 操作偶數次時，不包含 uuu 的最大值；而 f(u,1)f(u, 1)f(u,1) 表示在 uuu 的子樹中， uuu 操作奇數次時，不包含 uuu 的最大值。
    • 所以在不操作 (u,fa)(u, fa)(u,fa) 這條邊時，直接從 f(u,0)+nums[u]f(u, 0) + nums[u]f(u,0)+nums[u] 和 f(u,1)+(nums[u]⊕k)f(u, 1) + (nums[u] \oplus k)f(u,1)+(nums[u]⊕k) 中取最大值即可。
  • dfs(u,fa,1)=max⁡(f(u,1)+nums[v],f(u,0)+(nums[u]⊕k))dfs(u, fa, 1) = \max(f(u, 1) + nums[v], f(u, 0) + (nums[u] \oplus k))dfs(u,fa,1)=max(f(u,1)+nums[v],f(u,0)+(nums[u]⊕k))，表示操作 (u,fa)(u, fa)(u,fa) 這條邊。
    • 在 uuu 的子樹中操作奇數次，加上操作 (u,fa)(u, fa)(u,fa) 這條邊後，等同 uuu 操作偶數次，故貢獻為 nums[u]nums[u]nums[u] ；反之若在 uuu 的子樹中操作偶數次，加上操作 (u,fa)(u, fa)(u,fa) 這條邊後，等同 uuu 操作奇數次，故貢獻為 nums[u]⊕knums[u] \oplus knums[u]⊕k。
    • 所以在操作 (u,fa)(u, fa)(u,fa) 這條邊時，從 f(u,1)+nums[v]f(u, 1) + nums[v]f(u,1)+nums[v] 和 f(u,0)+(nums[u]⊕k)f(u, 0) + (nums[u] \oplus k)f(u,0)+(nums[u]⊕k) 中取最大值即可。

則 f(u,0)f(u, 0)f(u,0) 和 f(u,1)f(u, 1)f(u,1) 的轉移方程式如下：

  • f(u,0)=max⁡(f(u,0)+dfs(v,u,0),f(u,1)+dfs(v,u,1))f(u, 0) = \max(f(u, 0) + dfs(v, u, 0), f(u, 1) + dfs(v, u, 1))f(u,0)=max(f(u,0)+dfs(v,u,0),f(u,1)+dfs(v,u,1))，表示不操作 (u,v)(u, v)(u,v) 或操作 (u,v)(u, v)(u,v)。
  • f(u,1)=max⁡(f(u,1)+dfs(v,u,0),f(u,0)+dfs(v,u,1))f(u, 1) = \max(f(u, 1) + dfs(v, u, 0), f(u, 0) + dfs(v, u, 1))f(u,1)=max(f(u,1)+dfs(v,u,0),f(u,0)+dfs(v,u,1))，表示不操作 (u,v)(u, v)(u,v) 或操作 (u,v)(u, v)(u,v)。

在寫出所有遞迴關係後，我們可以用 DFS 的方式來遍歷整棵樹，並在遍歷的過程中更新 f(u,0)f(u, 0)f(u,0) 和 f(u,1)f(u, 1)f(u,1) 的值， dfsdfsdfs 函數返回 dfs(u,fa,0)dfs(u, fa, 0)dfs(u,fa,0) 和 dfs(u,fa,1)dfs(u, fa, 1)dfs(u,fa,1) 的值。

由於是無向樹，可以令根節點為 000，設其父節點為 −1-1−1 。由於根節點沒有父節點，顯然無法操作 (0,−1)(0, -1)(0,−1) 這條邊，故最後返回的值需為 dfs(0,−1,0)dfs(0, -1, 0)dfs(0,−1,0) ，而 dfs(0,−1,1)dfs(0, -1, 1)dfs(0,−1,1) 為非法的情況。

複雜度分析

  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為節點數。
  • 空間複雜度：O(n)O(n)O(n)。

方法二：狀態機DP

對於樹上的任兩點 u,vu, vu,v，樹上兩點間必存在一條路徑，而沿著這條路徑操作，可以將 uuu 和 vvv 之間的所有點(除了 uuu 和 vvv 之外)的操作次數 +2+2+2 ，但由於 XOR 的性質，操作 +2+2+2 次和操作 000 次是等價的，只有 uuu 和 vvv 會被真正操作各 111 次。故其實可以直接選兩點操作，不用建圖。

令 dp[i][0/1]dp[i][0/1]dp[i][0/1] 表示前 iii 個點操作偶數/奇數次時的最大值，則轉移方程式如下：

  • dp[i][0]=max⁡(dp[i−1][0]+nums[i],dp[i−1][1]+(nums[i]⊕k))dp[i][0] = \max(dp[i-1][0] + nums[i], dp[i-1][1] + (nums[i] \oplus k))dp[i][0]=max(dp[i−1][0]+nums[i],dp[i−1][1]+(nums[i]⊕k))
    前 iii 個點操作偶數次，可以從前 i−1i-1i−1 個點操作偶數次且當前點不操作，或者前 i−1i-1i−1 個點操作奇數次且當前點操作轉移而來，兩者取最大值。
  • dp[i][1]=max⁡(dp[i−1][1]+nums[i],dp[i−1][0]+(nums[i]⊕k))dp[i][1] = \max(dp[i-1][1] + nums[i], dp[i-1][0] + (nums[i] \oplus k))dp[i][1]=max(dp[i−1][1]+nums[i],dp[i−1][0]+(nums[i]⊕k))
    前 iii 個點操作奇數次，可以從前 i−1i-1i−1 個點操作奇數次且當前點不操作，或者前 i−1i-1i−1 個點操作偶數次且當前點操作轉移而來，兩者取最大值。

複雜度分析

  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為節點數，也可以說是 numsnumsnums 的長度。
  • 空間複雜度：O(n)O(n)O(n)。

方法二：狀態機DP，空間優化

由於轉移只和前一個狀態有關，故可以進一步優化空間為 O(1)O(1)O(1) 。

複雜度分析

  • 時間複雜度：O(n)O(n)O(n)。
  • 空間複雜度：O(1)O(1)O(1)。
