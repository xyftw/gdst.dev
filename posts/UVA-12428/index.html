🔗 UVA-12428 Enemy at the Gates

tags: 貪婪(Greedy), 二分搜尋(Binary Search)

範例程式碼已於UVA、瘋狂程設(CPE)、ZeroJudge 上皆測試通過。
python 可以於 UVA 和 瘋狂程設(CPE) 上直接通過；但於 ZeroJudge 上需要加上二分搜尋(Binary Search) 的技巧才能通過。

題意

  • 給定一個具有 mmm 條邊、nnn 個點的 簡單無向連通圖 。
  • 如果刪除一條邊後，會使得圖中的某些點無法連通，則這條邊被認為是重要/關鍵的，求給定的圖中最多有幾條這種邊？

思路：貪婪(Greedy) + 暴力枚舉(Brute Force) / 二分搜尋(Binary Search)

  • 首先來看只有一個關鍵邊的情況，如範例一的 (1,4)(1, 4)(1,4)，此時該條邊連接了一個 孤立點 和其餘點的 連通分量 。
  • 為使這種關鍵邊越多，我們可以將每條關鍵邊都視為一個孤立點和其餘點的連通分量連接，如範例二中可以視為 333 個孤立點和 111 個連通分量的連接。
  • 但此時有個問題，若存在 kkk 個關鍵邊，則代表存在 kkk 個孤立點，且由於原圖是連通的，因此剩下 n−kn-kn−k 個點要構成連通分量。為方便說明，以下令 x=n−kx = n-kx=n−k。
  • 由於給定的圖必須是簡單圖，不能存在重複的邊，因此這個連通分量中最多只能有 x×(x−1)2\frac{x \times (x-1)}{2}2x×(x−1)​ 條邊，因此若剩餘的邊數 m−k≤x×(x−1)2m-k \leq \frac{x \times (x-1)}{2}m−k≤2x×(x−1)​，則代表可以存在 kkk 個關鍵邊，反之則無法存在 kkk 個關鍵邊。

解法一：貪婪(Greedy) + 暴力枚舉(Brute Force)

  • 由於目標是找到最多的關鍵邊，因此我們可以從 k=n−1k = n-1k=n−1 開始枚舉，直到找到最大的 kkk 使得 x×(x−1)2≥m−k\frac{x \times (x-1)}{2} \geq m-k2x×(x−1)​≥m−k 即可。
    • 在枚舉 kkk 時有個小地方要注意，關鍵邊的數量最多是 n−1n-1n−1 條，而不是 nnn 條，可以自行思考為什麼。
  • 時間複雜度：O(n)O(n)O(n)，對於每個詢問。
  • 空間複雜度：O(1)O(1)O(1)。

解法二：貪婪(Greedy) + 二分搜尋(Binary Search)

由於 UVA 和 ZeroJudge 上使用的是相對較舊的 Python 版本，其中的 bisect 套件並不支持自訂 key function，因此需要自行實作二分搜尋(Binary Search)的過程。

  • 將上述的暴力枚舉上傳到 ZeroJudge 後，會發現使用 Python 的程式碼會超時，因此我們需要使用二分搜尋(Binary Search)的技巧來加速。
  • 要二分必須滿足單調性質，而上述暴力枚舉中的檢查條件 x×(x−1)2≥m−k\frac{x \times (x-1)}{2} \geq m - k2x×(x−1)​≥m−k 就是單調的，因此我們可以將其視為一個單調函數，進行二分搜尋。
  • 二分是紅藍染色，而這裡的單調性質是從 TrueTrueTrue (紅色) 到 FalseFalseFalse (藍色) ，而閉區間 [left,right][left, right][left,right] 的二分搜尋，最後的 rightrightright 會是最後一個紅色、leftleftleft 會是第一個藍色。由於我們要求的是最多的關鍵邊，因此最後的答案就是 rightrightright。
  • 時間複雜度：O(log⁡n)O(\log n)O(logn)，對於每個詢問。
  • 空間複雜度：O(1)O(1)O(1)。
