🟢 3184. Count Pairs That Form a Complete Day I _

tags: 暴力法(Brute Force) 雜湊表(Hash Table) 模運算(Modulo)

本題與下一題相同，只差在數字範圍，建議直接使用下一題的解法。

題意

給定一個整數陣列 hours\text{hours}hours。返回有多少對 (i,j)(i, j)(i,j) 滿足 i<ji < ji<j 且 hours[i]+hours[j]\text{hours}[i] + \text{hours}[j]hours[i]+hours[j] 為 242424 的整數倍數。

約束條件

  • 1≤n=hours.length≤1001 \leq n = \text{hours.length} \leq 1001≤n=hours.length≤100
  • 1≤hours[i]≤1091 \leq \text{hours}[i] \leq 10^91≤hours[i]≤109

思路：暴力法(Brute Force)

對於陣列中的每個元素 hours[i]\text{hours}[i]hours[i]，我們遍歷其後面的所有元素 hours[j]\text{hours}[j]hours[j]，其中 j>ij > ij>i。然後，我們計算這兩個數的和 hours[i]+hours[j]\text{hours}[i] + \text{hours}[j]hours[i]+hours[j] 是否是 242424 的整數倍數。如果符合條件，將答案加一。

由於 nnn 最大為 100100100，所以 O(n2)\mathcal{O}(n^2)O(n2) 的暴力法是可以接受的。關於如何優化，請參考下一題。

複雜度分析

  • 時間複雜度：O(n2)\mathcal{O}(n^2)O(n2) ，其中 nnn 為陣列的長度。
  • 空間複雜度：O(1)\mathcal{O}(1)O(1) 。

🟡 3185. Count Pairs That Form a Complete Day II _

tags: 雜湊表(Hash Table) 模運算(Modulo)

題意

給定一個整數陣列 hours\text{hours}hours。返回有多少對 (i,j)(i, j)(i,j) 滿足 i<ji < ji<j 且 hours[i]+hours[j]\text{hours}[i] + \text{hours}[j]hours[i]+hours[j] 為 242424 的整數倍數。

約束條件

  • 1≤n=hours.length≤5⋅1051 \leq n = \text{hours.length} \leq 5 \cdot 10^51≤n=hours.length≤5⋅105
  • 1≤hours[i]≤1091 \leq \text{hours}[i] \leq 10^91≤hours[i]≤109

思路：雜湊表(Hash Table) + 模運算(Modulo)

從上一題的暴力法中可以注意到，我們在考慮到 nums[i]nums[i]nums[i] 時，只需要知道 左側 的數字中有多少個數字 yyy 滿足 y+nums[i]≡0(mod24)y + nums[i] \equiv 0 \pmod{24}y+nums[i]≡0(mod24)。這樣我們就可以在 O(1)\mathcal{O}(1)O(1) 的時間內計算出答案。

因此可以使用類似於 1. Two Sum 的方法，使用 雜湊表(Hash Table) 來記錄每個數字的出現次數，然後對於每個數字 xxx，我們只需要查詢 24−(x mod 24)24 - (x \bmod{24})24−(xmod24) 的出現次數即可。

但需要注意到存在一種特殊情況，即 x=0x = 0x=0 的情況，此時 24−x=2424 - x = 2424−x=24，可以使用 特判 或 二次取模 的方式來處理。

具體步驟如下：

  1. 建立一個大小為 242424 的陣列 cnt\text{cnt}cnt 來記錄每個數字對 242424 取模後的出現次數。
  2. 遍歷 hours\text{hours}hours 陣列，對於每個元素 xxx：
    • 計算 x mod 24x \bmod{24}xmod24 (也就是小時數對 242424 取餘)
    • 如果 xxx 不為 000，則將 cnt[24−x]\text{cnt}[24 - x]cnt[24−x] 加到答案 ans\text{ans}ans 上(因為 hours[i]+hours[j]=24\text{hours}[i] + \text{hours}[j] = 24hours[i]+hours[j]=24 時，iii 和 jjj 就是一對)；如果 xxx 為 000，則將 cnt[0]\text{cnt}[0]cnt[0] 加到答案 ans\text{ans}ans 上。
    • 或是做二次取模，將 cnt[(24−x) mod 24]\text{cnt}[(24 - x) \bmod{24}]cnt[(24−x)mod24] 加到答案 ans\text{ans}ans 上。
    • 將 cnt[x]\text{cnt}[x]cnt[x] 加 111，更新計數。
  3. 最後返回答案 ans\text{ans}ans。

複雜度分析

  • 時間複雜度：O(n)\mathcal{O}(n)O(n) ，其中 nnn 為陣列的長度。
  • 空間複雜度：O(24)=O(1)\mathcal{O}(24) = \mathcal{O}(1)O(24)=O(1) 。

🟡 3186. Maximum Total Damage With Spell Casting _

tags: 動態規劃(DP) 記憶化搜索(Memoization) 打家劫舍

題意

一個魔法師能夠施放許多不同的魔法。

給定一個陣列 powerpowerpower，其中每個元素表示一個魔法的傷害值，可能有多個魔法的傷害值相同。

但當魔法師決定施放一個傷害值為 power[i]power[i]power[i] 的魔法時，就無法再施放傷害值為 power[i]−2power[i] - 2power[i]−2、power[i]−1power[i] - 1power[i]−1、power[i]+1power[i] + 1power[i]+1 或 power[i]+2power[i] + 2power[i]+2 的魔法。

每個魔法最多只能被施放 一次。

返回這個魔法師能夠施放的 最大 可能總傷害值。

思路：基於值域的打家劫舍動態規劃(DP)

這個問題可以轉換成值域上的問題，由於任兩個施放的魔法之間的差值需超過 222，因此可以統計每個傷害值的出現次數，並將其排序，這樣就可以將問題轉換成 打家劫舍 問題。

若選擇施放一個傷害為 xxx 的魔法，其他傷害值為 xxx 的魔法也可以施放，因此其貢獻為 x×cnt[x]x \times \text{cnt}[x]x×cnt[x]，其中 cnt[x]\text{cnt}[x]cnt[x] 表示傷害值為 xxx 的魔法的數量。

但由於任兩個施放的魔法之間的差值需超過 222，因此可以由小到大便利每個傷害值，對於每個傷害值 xxx，若前一個鍵值為 yyy，且 x−y>2x - y > 2x−y>2，則可以從前一個鍵值 yyy 轉移過來，否則只能繼續往前找能夠轉移的鍵值。但由於已經轉換為值域問題，因此每次轉移時，最多只要考慮前 333 個鍵值。

方法一：維護施放前 i 個魔法，且第 i 個魔法必須施放的最大總傷害

首先統計每個傷害值的出現次數，並將其鍵值排序，然後使用動態規劃來解決問題。

令 dp[i]dp[i]dp[i] 表示以施放前 iii 個魔法，且第 iii 個魔法必須施放的最大總傷害，則可以寫出以下轉移式：

  • dp[i]={max⁡0≤j<ikeys[i]−keys[j]>2dp[j]}+keys[i]×cnt[keys[i]]dp[i] = \{ \max\limits_{\substack{ 0 \leq j < i \\ \text{keys}[i] - \text{keys}[j] > 2}} dp[j] \} + \text{keys}[i] \times \text{cnt}[\text{keys}[i]]dp[i]={0≤j<ikeys[i]−keys[j]>2​max​dp[j]}+keys[i]×cnt[keys[i]]

但是這樣會有大量的轉移，因此可以維護一個陣列 mxmxmx，其中 mx[i]=max⁡0≤j≤idp[j]mx[i] = \max\limits_{0 \leq j \leq i} dp[j]mx[i]=0≤j≤imax​dp[j] ，這樣就可以將轉移的時間從 O(n2)\mathcal{O}(n^2)O(n2) 降為 O(n)\mathcal{O}(n)O(n)。

複雜度分析

  • 時間複雜度：O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) ，其中 nnn 為鍵值的數量，也可以視為 powerpowerpower 陣列的長度。
    • 統計每個傷害值的出現次數需要 O(n)\mathcal{O}(n)O(n) 時間。
    • 排序鍵值需要 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) 時間。
    • 動態規劃需要 O(n)\mathcal{O}(n)O(n) 時間，對於每個狀態，最多只要考慮三個轉移來源即可。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) 。

方法二：維護前 i 個魔法的最大總傷害

與方法一相同，首先統計每個傷害值的出現次數，並將其鍵值排序，然後使用動態規劃來解決問題。但我們改成維護一個隨着魔法數量增加而更新的狀態，即考慮在前 iii 個魔法中能夠達到的最大總傷害值。

定義 dp[i]dp[i]dp[i] 表示考慮到第 iii 個魔法時的最大總傷害，則可以寫出轉移式如下：

  • dp[i]=max⁡(dp[i−1],dp[j]+power[i]×cnt[power[i]])dp[i] = \max(dp[i-1], dp[j] + \text{power}[i] \times \text{cnt}[\text{power}[i]])dp[i]=max(dp[i−1],dp[j]+power[i]×cnt[power[i]]) ，其中 jjj 是最近的一個與第 iii 個魔法的傷害值相差超過 222 的魔法。
    • 不施放第 iii 個魔法，則 dp[i]=dp[i−1]dp[i] = dp[i-1]dp[i]=dp[i−1] 。
    • 施放第 iii 個魔法，則我們需要找到前面最近的一個與第 iii 個魔法的傷害值相差超過 222 的魔法 jjj ，此時 dp[i]=dp[j]+power[i]×cnt[power[i]]dp[i] = dp[j] + \text{power}[i] \times \text{cnt}[\text{power}[i]]dp[i]=dp[j]+power[i]×cnt[power[i]] 。
    • 兩者取最大值即可。
  • 最終答案即為 dp[m−1]dp[m-1]dp[m−1] ，其中 mmm 為傷害值的種類數。

這裡以 Top-down 記憶化搜索(Memoization) 的方式來實現，同樣可改成 Bottom-up 的方式。

複雜度分析

  • 時間複雜度：O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) ，其中 nnn 為鍵值的數量，也可以視為 powerpowerpower 陣列的長度。
    • 統計每個傷害值的出現次數需要 O(n)\mathcal{O}(n)O(n) 時間。
    • 排序鍵值需要 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) 時間。
    • 動態規劃需要 O(n)\mathcal{O}(n)O(n) 時間，對於每個 iii ，最多只要考慮前 333 個鍵值即可。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) 。

🔴 3187. Peaks in Array _

tags: 樹狀陣列(BIT) 線段樹(Segment Tree) 分治法(Divide and Conquer)

題意

定義一個陣列中的 峰值(Peak) 元素為 大於 前後相鄰元素的元素。

給定一個整數陣列 numsnumsnums 和一個二維整數陣列 queriesqueriesqueries。

你需要處理以下兩種類型的查詢：

  • queries[i]=[1,li,ri]queries[i] = [1, l_i, r_i]queries[i]=[1,li​,ri​]，確定子陣列 nums[li..ri]nums[l_i..r_i]nums[li​..ri​] 中的 峰值 元素的數量。
  • queries[i]=[2,indexi,vali]queries[i] = [2, index_i, val_i]queries[i]=[2,indexi​,vali​]，將 nums[indexi]nums[index_i]nums[indexi​] 變為 valival_ivali​。

返回一個陣列 answeransweranswer，依次包含每一個第一種查詢的答案。

注意：陣列或子陣列中的第一個和最後一個元素都 不能 是 峰值 元素。

思路：PURQ + 樹狀陣列(Binary Indexed Tree) / 線段樹(Segment Tree)

根據題意，我們需要做到 單點修改區間查詢(Point Update Range Query, PURQ) 的操作，為此可以使用 樹狀陣列(Binary Indexed Tree) 或 線段樹(Segment Tree) 來解決。

接下來就是要如何維護區間內的 峰值 元素的數量，這裡有兩種思路

  1. 維護峰值的位置，轉換為 區間求和 問題
  2. 維護區間內的峰值數量，轉換為 區間分治 問題。

方法一：維護峰值，轉換為區間求和問題

首先遍歷一次陣列，將每個峰值的位置記錄下來，然後使用 樹狀陣列(Binary Indexed Tree) 或 線段樹(Segment Tree) 來維護區間內的峰值數量，這裡使用樹狀陣列(Binary Indexed Tree) 來解決，關於其細節可以參考 树状数组从入门到下车 。

關於修改和查詢操作的說明如下：

  • 進行單點修改時，由於會變動的只有修改點以及其左右兩個點，總共三個位置，因此可以透過更新這三個位置是否為峰值，來更新區間內的峰值數量。
    • 更新時可以先檢查原本是否為峰值再進行修改，以避免修改前後的狀態相同，造成不必要的更新。
  • 進行區間查詢時，由於子陣列的頭尾元素不能是峰值，因此查詢時不能包含區間的頭尾，所以要查詢的區間為 [l+1,r−1][l+1, r-1][l+1,r−1]。
    • 但需注意當 l+1>r−1l+1 > r-1l+1>r−1 時，為空區間，此時應該返回 000。

複雜度分析

  • 時間複雜度：O(n+qlog⁡n)\mathcal{O}(n + q \log n)O(n+qlogn) ，其中 nnn 為陣列的長度，qqq 為查詢的個數。
    • 初始化 樹狀陣列(Binary Indexed Tree) 的時間複雜度為 O(n)\mathcal{O}(n)O(n) 。
    • 單點修改的時間複雜度為 O(log⁡n)\mathcal{O}(\log n)O(logn) 。
    • 區間查詢的時間複雜度為 O(log⁡n)\mathcal{O}(\log n)O(logn) 。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) ，不計算答案的空間。

方法二：以區間分治的方式維護峰值數量

在這個方法中，我們使用 線段樹(Segment Tree) 以區間合併的方式來維護區間內的峰值數量。為此需要定義合併函數 merge，該函數根據左右兩個子節點的結果計算出合併後的節點結果，合併方法如下：

  • 兩個區間合併時，會新增的峰值為左區間的最後一個元素和右區間的第一個元素。
    • 但由於這兩個元素相鄰，因此最多只有一個是峰值，因此可以使用 or 來合併。
  • 透過檢查這兩個元素是否為峰值，來更新合併後的區間的峰值數量。
  • 由於只要檢查兩個元素，因此 merge 可以在 O(1)\mathcal{O}(1)O(1) 的時間內完成。

使用線段樹維護若干區間的峰值數量，可以在 O(log⁡n)\mathcal{O}(\log n)O(logn) 的時間內完成區間的查詢和修改操作，具體細節可參考 线段树从入门到急停 。程式碼的部分大部分都是模板，需要修改的地方主要是合併函數 merge 的部分。

複雜度分析

  • 時間複雜度：O(nlog⁡n+qlog⁡n)\mathcal{O}(n \log n + q \log n)O(nlogn+qlogn) ，其中 nnn 為陣列的長度，qqq 為查詢的個數。
    • 初始化 線段樹(Segment Tree) 的時間複雜度為 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) 。
    • 單點修改的時間複雜度為 O(log⁡n)\mathcal{O}(\log n)O(logn) 。
    • 區間查詢的時間複雜度為 O(log⁡n)\mathcal{O}(\log n)O(logn) 。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) ，不計算答案的空間。
