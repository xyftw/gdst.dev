🔗 🟡 2597. The Number of Beautiful Subsets 2023

tags: Weekly Contest 337 回溯(Backtracking) 剪枝(Pruning) 子集型回溯 動態規劃(Dynamic Programming) 雜湊表(Hash Table) 數學(Math) 乘法原理

題意

給定一個正整數陣列 numsnumsnums 和一個正整數 kkk。

如果 numsnumsnums 的子集中，任意兩個整數的絕對差值不等於 kkk，則該子集是一個 美麗(beautiful) 子集。

返回陣列 numsnumsnums 中 非空 且 美麗 的子集的數量。

限制

  • 1≤nums.length≤201 \leq nums.length \leq 201≤nums.length≤20
  • 1≤nums[i],k≤10001 \leq nums[i], k \leq 10001≤nums[i],k≤1000

思路

解法一：回溯(Backtracking) + 剪枝(Pruning)

要構造出所有可能的子集，可以使用回溯法，類似 78. Subsets，但是需要加入條件判斷。

對於遍歷到的每個元素 x=nums[i]x = nums[i]x=nums[i] ，都有兩種可能：在當前子集中或不在當前子集中。但若 nums[i]±knums[i] \pm knums[i]±k 在子集中，則不能選擇 xxx。為此我們需要維護當前在子集中的元素數量，可以使用雜湊表或陣列 cntcntcnt 來記錄。

  • 若不選擇 xxx，則直接遞迴到下一個元素，即 dfs(i+1)dfs(i+1)dfs(i+1)。
  • 若選擇 xxx，則需要檢查 cnt[x−k]cnt[x-k]cnt[x−k] 和 cnt[x+k]cnt[x+k]cnt[x+k] 是否為 000，若不為 000 則不能選擇 xxx。若滿足條件，需要將 xxx 加入到子集中，即累加 cnt[x]cnt[x]cnt[x]，再遞迴到下一個元素，遞迴結束後，需要將 xxx 從子集中移除，即減去 cnt[x]cnt[x]cnt[x]。

根據題意，會使用到的索引範圍在 [−k,mx+k][-k, mx+k][−k,mx+k] 之內，其中 mxmxmx 為陣列中的最大值。或是根據限制條件中 1≤nums[i],k≤10001 \leq nums[i], k \leq 10001≤nums[i],k≤1000，因此值域應為 [−999,2000][-999, 2000][−999,2000] ，可以使用一個大小為 300030003000 的陣列 cntcntcnt 來記錄。但需要注意負數的情況：

  • 在 Python 中，若索引為負數，則會從陣列的最後一個元素開始，因此只要確保後 kkk 個索引不會被使用即可。
  • 在 C++ 中，若索引為負數，則會造成陣列越界，因此需要將索引加上 kkk 來避免負數索引。

而子集型回溯有兩種寫法：

  1. 選或不選當前元素，當 iii 等於陣列的長度時，表示已經遍歷完所有元素，此時累加答案。
  2. 枚舉下一個選的元素 jjj，此時每個遞迴都恰好是一個子集，因此直接在遞迴開始時累加答案即可。

在選擇時直接根據 xxx 的值來判斷是否選擇，可以減少一些不必要的遞迴，即為 剪枝(Pruning)。

複雜度分析

  • 時間複雜度：O(2n)\mathcal{O}(2^n)O(2n)，每個元素都有 222 種可能，共有 nnn 個元素，因此共有 2n2^n2n 種可能。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n)，遞迴的深度為 O(n)O(n)O(n)，若使用雜湊表，cntcntcnt 的空間複雜度為 O(n)O(n)O(n)；若使用陣列，cntcntcnt 的空間複雜度為 O(n+2k)O(n+2k)O(n+2k) ，本題中仍然可視為 O(n)O(n)O(n)。

寫法一：選或不選當前元素

寫法二：枚舉下一個選擇

解法二：動態規劃(Dynamic Programming) + 乘法原理

首先引入一個前提，兩個整數要相差 kkk，則必須滿足 x≡y(modk)x \equiv y \pmod{k}x≡y(modk)，因此可以將陣列中的整數分組，每個組中的整數 xxx 滿足 x≡i(modk)x \equiv i \pmod{k}x≡i(modk)，其中 0≤i<k0 \leq i < k0≤i<k，不同組內的選擇不會影響到彼此，因此可以分開計算，再使用 乘法原理 將結果相乘。

對於每組內的整數，可以使用 動態規劃 來計算方案數，令 dp[i]dp[i]dp[i] 表示前 iii 個整數的方案數，注意這裡的 iii 是 111-indexed。對於每個整數 xxx，都有 2cx2^{c_x}2cx​ 種選擇，其中 cx=cnt[x]c_x = cnt[x]cx​=cnt[x] 表示整數 xxx 在陣列中出現的次數，在這些選擇中，選 xxx 的方案數有 2cx−12^{c_x} - 12cx​−1 種，不選 xxx 的方案數有 111 種。但是若 xix_ixi​ 和前一個整數 xi−1x_{i-1}xi−1​ 相差 kkk，則不能同時選擇 xix_ixi​ 和 xi−1x_{i-1}xi−1​，因此需要分情況討論：

  • 初始化 dp[0]=1dp[0] = 1dp[0]=1 和 dp[1]=2cx1dp[1] = 2^{c_{x_1}}dp[1]=2cx1​​，其中 x1x_1x1​ 為第一個整數。
  • 若 xix_ixi​ 和 xi−1x_{i-1}xi−1​ 相差 kkk，則 xix_ixi​ 的方案數 dp[i]=dp[i−1]+dp[i−2]×(2cxi−1)dp[i] = dp[i-1] + dp[i-2] \times (2^{c_{x_i}} - 1)dp[i]=dp[i−1]+dp[i−2]×(2cxi​​−1)。
    • 不選 xix_ixi​ ，此時可選或可不選 xi−1x_{i-1}xi−1​，此時方案數為 dp[i−1]dp[i-1]dp[i−1]。
    • 若選 xix_ixi​，則不能選 xi−1x_{i-1}xi−1​，而選 xix_ixi​ 的方案數為 2cxi−12^{c_{x_i}} - 12cxi​​−1 ，此時方案數為 dp[i−2]×(2cxi−1)dp[i-2] \times (2^{c_{x_i}} - 1)dp[i−2]×(2cxi​​−1)。
  • 若 xix_ixi​ 和 xi−1x_{i-1}xi−1​ 不相差 kkk，則 xix_ixi​ 的方案數 dp[i]=dp[i−1]×2cxidp[i] = dp[i-1] \times 2^{c_{x_i}}dp[i]=dp[i−1]×2cxi​​ ，代表可選或可不選 xix_ixi​。

得每組內的方案數為 dp[m]dp[m]dp[m]，其中 mmm 為組內整數的個數，再將所有組的方案數相乘，減去空集合後即為答案。

複雜度分析

  • 時間複雜度：O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn)，看似為 O(n2)O(n^2)O(n2) 但其實每個整數只會被分到一組內，因此考慮每組內排序的時間複雜度即可，為 O(mlog⁡m)O(m \log m)O(mlogm)，其中 mmm 為組內整數的最大個數，或可視為 O(nlog⁡n)O(n \log n)O(nlogn)。
  • 空間複雜度：O(n)\mathcal{O}(n)O(n)，使用雜湊表來記錄每組整數的出現次數，空間複雜度為 O(n)O(n)O(n)。
