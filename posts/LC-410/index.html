🔗 🔴 410. Split Array Largest Sum

tags: 動態規劃(DP) 記憶化搜尋(Memoization) 前綴和(Prefix Sum) 二分搜尋(Binary Search)

題意：最小化最大值

給定一個非負整數陣列 numsnumsnums 和一個整數 kkk，你需要將這個陣列分成 kkk 個非空的連續子陣列，使得 最大子陣列和 能夠被 最小化 。

返回分割後 最大子陣列和 的 最小值。

約束條件

  • 1≤nums.length≤10001 \leq \text{nums.length} \leq 10001≤nums.length≤1000
  • 0≤nums[i]≤1060 \leq \text{nums}[i] \leq 10^60≤nums[i]≤106
  • 1≤k≤min⁡(50,nums.length)1 \leq k \leq \min(50, \text{nums.length})1≤k≤min(50,nums.length)

思路

為求出每個子陣列的和，我們可以使用 前綴和(Prefix Sum) 來預先計算出陣列的部分和，這樣可以在 O(1)O(1)O(1) 時間內獲得任意區間的和。令 s[i]s[i]s[i] 表示前 iii 個數字的和，則 s[i]=nums[0]+nums[1]+…+nums[i−1]=∑j=0i−1nums[j]s[i] = \text{nums}[0] + \text{nums}[1] + \ldots + \text{nums}[i-1] = \displaystyle\sum_{j=0}^{i-1} \text{nums}[j]s[i]=nums[0]+nums[1]+…+nums[i−1]=j=0∑i−1​nums[j] ，則求 [l,r][l, r][l,r] 區間的和，只需要計算 s[r+1]−s[l]s[r+1] - s[l]s[r+1]−s[l] 即可。

根據題意，子陣列的最大值為 ∑nums\sum \text{nums}∑nums ，最小值為 max⁡nums\max \text{nums}maxnums 。而根據約束條件， ∑nums≤106×1000=109\sum \text{nums} \leq 10^6 \times 1000 = 10^9∑nums≤106×1000=109 ， max⁡nums≤106\max \text{nums} \leq 10^6maxnums≤106 。

方法一：Top-down DP / 記憶化搜尋(Memoization)

令 f(i,j)f(i, j)f(i,j) 表示將前 iii 個數字分成 jjj 個子陣列，所得到的最大和的最小值。則可以透過考慮上一段子陣列的結尾位置 xxx，將問題轉換為子問題 f(x,j−1)f(x, j-1)f(x,j−1) ，此時的 最大子陣列和 為 max⁡(f(x,j−1),s[i]−s[x])\max(f(x, j-1), s[i] - s[x])max(f(x,j−1),s[i]−s[x]) ，即子問題的 最大子陣列和 最小值、以及當前子陣列的總和，兩者的最大值。並可以將其作為答案的候選值，在所有可能的 xxx 中，取最小值即可。

再來考慮一些細節和邊界情況：

  • 對於轉移來源 xxx 的範圍，由於子陣列必須為非空，因此前一段的結尾位置 xxx 必須滿足 0≤x<i0 \leq x < i0≤x<i 。
  • 當 j=1j = 1j=1 時，表示只有一個子陣列，此時可以直接返回前 iii 個數字的和 s[i]s[i]s[i] ；
  • 當 j>ij > ij>i 時，由於每個子陣列需要至少包含一個數字，因此無法分成 jjj 個子陣列，故此時返回一個足夠大的值，表示無法作為答案。

由於遞迴函數中存在大量重複計算的子問題，因此可以使用 記憶化搜尋(Memoization) 來保存計算結果，避免重複計算。

最後返回 f(n,k)f(n, k)f(n,k) 即可，其中 nnn 為陣列的長度，表示考慮所有數字，分成 kkk 個子陣列時，得到的 最大子陣列和 的最小值。

複雜度分析

  • 時間複雜度：O(n2⋅k)O(n^2 \cdot k)O(n2⋅k) ，其中 nnn 為陣列 numsnumsnums 的長度，kkk 為分割成的子陣列個數。
  • 空間複雜度：O(n⋅k)O(n \cdot k)O(n⋅k) ，記憶化搜索需要的空間。

若使用 functools.cache 進行記憶化搜索，可能需要一點運氣才能通過，可以多提交幾次試試看。

方法二：Bottom-up DP

和方法一同樣的方式，只是將遞迴函數改為迭代，以 Bottom-up 的方式計算所有子問題的答案。類似地，我們可以使用一個二維陣列 dpdpdp 來保存計算結果，其中 dp[i][j]dp[i][j]dp[i][j] 表示將前 iii 個數字分成 jjj 個子陣列，所得到的最大和的最小值。

  • 初始化：dp[0][0]=0dp[0][0] = 0dp[0][0]=0 ，表示前 000 個數字分成 000 個子陣列，所得到的最大和的最小值為 000 ；對於其他位置，初始化為一個足夠大的值。
  • 狀態轉移：對於每個 iii 和 jjj ，可以枚舉上一段的結尾位置 xxx ，並計算出子問題中 最大子陣列和 的最小值、以及當前子陣列的總和，兩者的最大值，在其中取最小值即可。因此，狀態轉移方程為 dp[i][j]=min⁡x=0i−1(max⁡(dp[x][j−1],s[i]−s[x]))dp[i][j] = \displaystyle\min_{x=0}^{i-1}(\max(dp[x][j-1], s[i] - s[x]))dp[i][j]=x=0mini−1​(max(dp[x][j−1],s[i]−s[x])) 。

使用三重循環來計算所有子問題的答案，最後返回 dp[n][k]dp[n][k]dp[n][k] 即可。需要注意，在確定 iii 後，jjj 的範圍需要在 [1,min⁡(i,k)][1, \min(i, k)][1,min(i,k)] 之間，否則無法將前 iii 個數字分成 jjj 個子陣列。

複雜度分析

  • 時間複雜度：O(n2⋅k)O(n^2 \cdot k)O(n2⋅k) ，其中 nnn 為陣列 numsnumsnums 的長度，kkk 為分割成的子陣列個數。
  • 空間複雜度：O(n⋅k)O(n \cdot k)O(n⋅k) ，動態規劃表格 dpdpdp 需要的空間。

方法三：二分搜尋(Binary Search)

雖然我們可以通過動態規劃的方式來解決這個問題，但是我們也能將問題轉換為判斷問題，即「是否可以將陣列分成 kkk 個子陣列，使得每個子陣列的和都小於等於 ddd」，如此便可以使用 二分搜尋(Binary Search) 來解決。

定義一個函數 check(d) 來檢查是否可以將陣列分成 kkk 個子陣列，使得每個子陣列的和都小於等於 ddd。

  • 利用 cntcntcnt 來記錄 子陣列和 ≤d\leq d≤d 的子陣列個數，sss 來記錄當前子陣列的和。
  • 遍歷陣列 numsnumsnums 中的每個元素 xxx ，如果 s+x>ds + x > ds+x>d ，則表示需要開始一個新的子陣列，並將 sss 設為 xxx ；否則，將 sss 加上 xxx 。
  • 最後返回 cnt≤kcnt \leq kcnt≤k ，即是否可以將陣列分成 kkk 個子陣列，使得每個子陣列的和都小於等於 ddd 。
    • 若可以分成 k−1k-1k−1 個 子陣列和 都 ≤d\leq d≤d 的子陣列，則可以將其中某個子陣列再分成兩個子陣列，直到湊滿 kkk 個子陣列為止，因此返回 cnt≤kcnt \leq kcnt≤k 。
    • 之所以能夠這樣檢查，是因為題目確保了 k≤nums.lengthk \leq \text{nums.length}k≤nums.length ，即 kkk 不會超過陣列的長度，故一定可以分成 kkk 個子陣列。

這裡使用左閉右閉區間進行二分搜尋，令 left=max⁡numsleft = \max \text{nums}left=maxnums ，right=∑numsright = \sum \text{nums}right=∑nums ，在 [left,right][left, right][left,right] 的範圍內進行二分搜尋，不斷調整左右範圍直到找到最大的最小價格差為止。

  • 根據定義的 check 函數，區間的左側為 False\text{False}False ，而右側為 True\text{True}True。
  • 如果 check(mid)=True\text{check}(mid) = \text{True}check(mid)=True，則將右範圍縮小到 [left,mid−1][left, mid-1][left,mid−1]。
  • 如果 check(mid)=False\text{check}(mid) = \text{False}check(mid)=False，則將左範圍擴大到 [mid+1,right][mid+1, right][mid+1,right]。
  • 當區間為空，即 left>rightleft > rightleft>right 時，終止搜尋。

最終返回 leftleftleft ，即 最小化 的 最大子陣列和。

複雜度分析

  • 時間複雜度：O(nlog⁡s)O(n \log s)O(nlogs) ，其中 sss 表示陣列 numsnumsnums 中所有元素的和。
    • 每次二分搜尋時，需要對陣列進行一次遍歷，時間複雜度為 O(n)O(n)O(n) ，因此總時間複雜度為 O(nlog⁡s)O(n \log s)O(nlogs) 。
  • 空間複雜度：O(1)O(1)O(1) 。
