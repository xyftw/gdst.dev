🔗 UVA-941 Permutations

tags: 數學(Math) 遞迴(Recursion) 回溯(Backtracking)

範例程式碼已於UVA、瘋狂程設(CPE)、ZeroJudge 上皆測試通過。

題意

給定一個字串 SSS 和一個整數 NNN，找到字串 SSS 的所有排列中的第 N+1N+1N+1 個最小元素（按照字典序）。這些排列是對字串的所有可能組合，並且按照字典序排序。

注意，給定的 SSS 可能沒有事先排序。

約束

  • 1≤∣S∣≤201 \leq |S| \leq 201≤∣S∣≤20
  • 1≤N+1≤20!1 \leq N + 1 \leq 20!1≤N+1≤20!

思路：數學(Math) + 縮小問題範圍

由於 S≤20S \leq 20S≤20，而 20!=243290200817664000020! = 243290200817664000020!=2432902008176640000，顯然生成所有排列不是一個好方法。因此應該使用數學方法計算排列的位置，而不是生成所有排列。

舉個🌰，假設 S="abcde"S = \rm{"abcde"}S="abcde"、 N=15N = 15N=15，改成用 0−indexed0\rm{-indexed}0−indexed 表示。

  • 由於需要按照字典序排序，故前 [0,23][0, 23][0,23] 個排列的第 111 個元素都是 aaa；前 [24,47][24, 47][24,47] 個排列的第 111 個元素都是 bbb，以此類推。而在確定了第一個元素後，也能按照相同的方法繼續確定之後的元素，但後續就不用考慮已經確定的字元。
  • 確定答案的第 111 個元素為 "a"\rm{"a"}"a" 後，剩餘的元素為 "bcde"\rm{"bcde"}"bcde"。且 N=15mod  4!=15N = 15 \mod 4! = 15N=15mod4!=15 ，故接下來是找到在 "bcde"\rm{"bcde"}"bcde" 的排列中的第 151515 個排列。
  • 在"bcde"\rm{"bcde"}"bcde" 的排列中，前 [0,5][0, 5][0,5] 個排列的第 111 個元素都是 bbb；前 [6,11][6, 11][6,11] 個排列的第 111 個元素都是 ccc；前 [12,17][12, 17][12,17] 個排列的第 111 個元素都是 ddd；前 [18,23][18, 23][18,23] 個排列的第 111 個元素都是 eee。由於 N=15N = 15N=15 ，因此答案的第 222 個元素是 ddd ，剩餘的元素為 "bce"\rm{"bce"}"bce"。且 N=15mod  3!=3N = 15 \mod 3! = 3N=15mod3!=3 ，故接下來是找到在 "bce"\rm{"bce"}"bce" 的排列中的第 333 個排列。
  • 在 "bce"\rm{"bce"}"bce" 的排列中，前 [0,1][0, 1][0,1] 個排列的第 111 個元素都是 bbb；前 [2,3][2, 3][2,3] 個排列的第 111 個元素都是 ccc；前 [4,5][4, 5][4,5] 個排列的第 111 個元素都是 eee。由於 N=3N = 3N=3 ，因此答案的第 333 個元素是 ccc ，剩餘的元素為 "be"\rm{"be"}"be"。且 N=3mod  2!=1N = 3 \mod 2! = 1N=3mod2!=1 ，故接下來是找到在 "be"\rm{"be"}"be" 的排列中的第 111 個排列。
  • 在 "be"\rm{"be"}"be" 的排列中，前 [0,0][0, 0][0,0] 個排列的第 111 個元素都是 bbb；前 [1,1][1, 1][1,1] 個排列的第 111 個元素都是 eee。由於 N=1N = 1N=1 ，因此答案的第 444 個元素是 eee ，剩餘的元素為 "b"\rm{"b"}"b"。且 N=1mod  1!=0N = 1 \mod 1! = 0N=1mod1!=0 。
  • 由於只剩下一個元素，故答案的第 555 個元素是 bbb。答案為 "adceb"\rm{"adceb"}"adceb"。

理解上述範例後，可以將其拆解成以下步驟：

  1. 將 SSS 排序
  2. 逐位確定每個元素，由於相同字元不能重複使用，因此每次確定一個元素後，要將其從 SSS 中刪除
    • 透過計算階乘來計算每個元素的位置
    • 透過取餘數來計算下一個元素的位置
  3. 重複步驟 222 直到 SSS 為空

複雜度分析

  • 時間複雜度：O(m2)\mathcal{O}(m^2)O(m2) ，其中 mmm 為 SSS 的長度。
    • 計算 factorial\rm{factorial}factorial 的時間複雜度為 O(m)\mathcal{O}(m)O(m) 。
    • 排序 SSS 的時間複雜度為 O(mlog⁡m)\mathcal{O}(m \log m)O(mlogm) 。
    • 每次確定元素時，需要刪除被確定的元素，需要 O(m)\mathcal{O}(m)O(m) 的時間，因此總共的時間複雜度為 O(m2)\mathcal{O}(m^2)O(m2) 。
  • 空間複雜度：O(m)\mathcal{O}(m)O(m)

類題

  • 🔴 60. Permutation Sequence
    基本上完全一樣，只是第幾個排列是從 111 開始計算，而不是 000。
