🔗 🟡 1482. Minimum Number of Days to Make m Bouquets 1946

tags: Week Contest 193 二分搜尋(Binary Search)

題意

給定一個長度為 nnn 的整數陣列 bloomDay\text{bloomDay}bloomDay，一個整數 mmm 和一個整數 kkk。

花園中有 nnn 朵花，第 iii 朵花將在 bloomDay[i]\text{bloomDay}[i]bloomDay[i] 開花。

現在需要製作 mmm 個花束，製作一個花束時，需要使用花園中 相鄰 的 kkk 朵花。

返回從花園中製作 mmm 個花束需要等待的最少天數。如果無法製作 mmm 個花束則返回 −1-1−1。

約束條件

  • bloomDay.length=n\text{bloomDay.length} = nbloomDay.length=n
  • 1≤n≤1051 \leq n \leq 10^51≤n≤105
  • 1≤bloomDay[i]≤1091 \leq \text{bloomDay}[i] \leq 10^91≤bloomDay[i]≤109
  • 1≤m≤1061 \leq m \leq 10^61≤m≤106
  • 1≤k≤n1 \leq k \leq n1≤k≤n

思路：二分搜尋(Binary Search)

首先確定思路，顯然要直接計算出最少需要等待的天數是很困難的，但對於給定的天數 ddd ，我們可以確定是否可以在 ddd 天內製作 mmm 個花束，因此可以使用 二分搜尋(Binary Search) 來找到最少需要等待的天數。

定義一個函數 check(d) 來檢查在待定天數 ddd 內是否能製作出 mmm 個花束，每束花需要使用相鄰的 kkk 朵花。

  • 使用 resresres 來記錄製作的花束數量，curcurcur 來記錄相鄰花的數量。
  • 遍歷 bloomDay\text{bloomDay}bloomDay 陣列中的每個元素 bloom\text{bloom}bloom
    • 如果 bloom≤d\text{bloom} \leq dbloom≤d，代表這朵花可以被使用，則 curcurcur 加一。如果 cur=kcur = kcur=k，代表有 kkk 朵相鄰花可以製作花束，則 resresres 加一，curcurcur 重置為 000。
    • 如果 bloomDay[i]>d\text{bloomDay}[i] > dbloomDay[i]>d，則代表這朵花在 ddd 天時還沒開花，由於使用的花必須是相鄰的，因此需要把相鄰花的數量重置為 000。
  • 最後返回 res≥mres \geq mres≥m，即是否可以製作 mmm 個花束。

這裡使用左閉右閉區間進行二分搜尋，令 left=min⁡(bloomDay)left = \min(\text{bloomDay})left=min(bloomDay)，right=max⁡(bloomDay)right = \max(\text{bloomDay})right=max(bloomDay)，在 [left,right][left, right][left,right] 的範圍內進行二分搜尋，不斷調整左右範圍直到找到最小天數為止。

  • 根據定義的 check 函數，區間的左側為 False\text{False}False ，而右側為 True\text{True}True。
  • 如果 check(mid)=True\text{check}(mid) = \text{True}check(mid)=True，則將右範圍縮小到 [left,mid−1][left, mid-1][left,mid−1]。
  • 如果 check(mid)=False\text{check}(mid) = \text{False}check(mid)=False，則將左範圍擴大到 [mid+1,right][mid+1, right][mid+1,right]。
  • 當區間為空，即 left>rightleft > rightleft>right 時，終止搜尋。

最終返回 leftleftleft，即第一個 check 函數返回 False\text{False}False 的天數，如果 left>max⁡(bloomDay)left > \max(\text{bloomDay})left>max(bloomDay)，則表示無法製作 mmm 個花束，返回 −1-1−1。

複雜度分析

  • 時間複雜度：O(nlog⁡M)\mathcal{O}(n \log M)O(nlogM)。其中 n=∣bloomDay∣n = |\text{bloomDay}|n=∣bloomDay∣，M=max⁡(bloomDay)M = \max(\text{bloomDay})M=max(bloomDay)，我們需要進行 log⁡M\log MlogM 次二分搜尋。每次搜尋需要遍歷 nnn 個元素來檢查是否可以製作 mmm 個花束。
  • 空間複雜度：O(1)\mathcal{O}(1)O(1)。
