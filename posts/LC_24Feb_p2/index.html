🟢 169. Majority Element

題意

  • 給定一個大小為 nnn 的一維陣列 numsnumsnums ，返回其中的多數元素(Majority Element)。
    • __多數元素(Majority Element)++ 是指在陣列中出現次數 大於 ⌊n2⌋\lfloor \frac{n}{2} \rfloor⌊2n​⌋ 的元素。
  • 給定的陣列是非空的，且總是存在多數元素。

思路：雜湊表(Hash Table) / Boyer-Moore Voting Algorithm

方法一：雜湊表(Hash Table)

  • 統計每個元素出現的次數，若統計時發現某個元素出現次數大於 ⌊n2⌋\lfloor \frac{n}{2} \rfloor⌊2n​⌋，則返回該元素。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為陣列的長度。
  • 空間複雜度：O(n)O(n)O(n)。

方法二：Boyer-Moore Voting Algorithm

  • 如果把主要元素計為 +1+1+1，把其他元素計為 −1-1−1，將它們全部加起來，顯然和大於 000。
  • 根據這個思路，我們可以維護一個主要元素的候選人 candidatecandidatecandidate 和其出現的次數 cntcntcnt。
  • 遍歷 numsnumsnums 中的所有元素，對於每個元素 xxx ，若 cntcntcnt 的值為 000，則將 xxx 的值賦予 candidatecandidatecandidate ，隨後判斷 xxx：
    • 若 xxx 與 candidatecandidatecandidate 相等，則計數器 cntcntcnt 的值增加 111；
    • 若 xxx 與 candidatecandidatecandidate 不等，則計數器 cntcntcnt 的值減少 111。
  • 當 cntcntcnt 的值為 000 時，存在兩種情況：
    • candidatecandidatecandidate 不是多數元素，則顯然需要嘗試選擇 xxx 作為新的候選人；
    • candidatecandidatecandidate 是多數元素，則捨去 xxx 之前的所有元素，candidatecandidatecandidate 仍然是多數元素，最終仍會被選為候選人，不影響結果。
  • 最終的 candidatecandidatecandidate 即為整個陣列的多數元素。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為陣列的長度。
  • 空間複雜度：O(1)O(1)O(1)。

🔴 987. Vertical Order Traversal of a Binary Tree 1676

題意

  • 給定一個Binary Tree的根節點 rootrootroot ，返回其 垂序遍歷(Vertical Order Traversal) 。
  • 令 rootrootroot 位於 (0,0)(0, 0)(0,0) ，且對於每個位於 (row,col)(row, col)(row,col) 的節點而言，其左右子節點分別位於 (row+1,col−1)(row + 1, col - 1)(row+1,col−1) 和 (row+1,col+1)(row + 1, col + 1)(row+1,col+1)。
  • 垂序遍歷(Vertical Order Traversal) 是從最左邊的column開始，至最右邊的column結束，從上到下的順序列表。
    • 相同位置上可能存在多個節點，在這種情況下，需要按照節點的值由小到大進行排序。

思路：BFS/DFS + 雜湊表(Hash Table)

  • 首先以BFS或DFS紀錄每個節點的位置，並使用Hash Table將相同column的節點存放在同一個list中，保存 (row,val)(row, val)(row,val) 的tuple，最後對每個column的list的節點進行排序即可。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為二叉樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)。

🟢 231. Power of Two

題意

  • 給定一個整數 nnn ，判斷其是否是 222 的冪次方。 如果是，返回 true ；否則，返回 false 。
  • 如果存在一個整數 xxx 使得 n==2xn == 2^xn==2x ，則認為 nnn 是 222 的冪次方。

思路：位運算(Bit Manipulation)

  • 在二進制表示中，222 的冪次方的數字只有一個 111 ，例如 23=82^3 = 823=8 的二進制表示為 1000 ，24=162^4 = 1624=16 的二進制表示為 10000 。
  • 因此我們可以判斷最低位的 111 是否為唯一的 111 即可，可以使用以下兩種方法：
    • n&(n−1)n \And (n-1)n&(n−1) 可以消除最低位的 111 ，如果 nnn 是 222 的冪次方，則 n&(n−1)=0n \And (n-1) = 0n&(n−1)=0 。
    • n&−nn \And -nn&−n 可以得到最低位的 111 ，如果 nnn 是 222 的冪次方，則 n&−n=nn \And -n = nn&−n=n 。
  • 時間複雜度：O(1)O(1)O(1)。
  • 空間複雜度：O(1)O(1)O(1)。

      • 右子樹：preorder[idx+1:]preorder[idx+1:]preorder[idx+1:] 和 inorder[idx+1:]inorder[idx+1:]inorder[idx+1:]

🟢 268. Missing Number

題意

  • 給定一個包含 [0,n][0, n][0,n] 中 nnn 個數的陣列 numsnumsnums ，返回 [0,n][0, n][0,n] 這個範圍內 沒有出現 在陣列中的那個數。

思路：數學(Math) / 排序(Sorting) / 雜湊表(Hash Table) / 位運算(Bit Manipulation)

方法一：數學(Math)

  • 計算出 [0,n][0, n][0,n] 的總和，然後減去陣列中所有數的總和，即可得到缺失的數。
  • 時間複雜度：O(n)O(n)O(n)。
  • 空間複雜度：O(1)O(1)O(1)。

方法二：排序(Sorting)

  • 將陣列排序，然後遍歷陣列，找到缺失的數。
  • 時間複雜度：O(nlog⁡n)O(n \log n)O(nlogn)。
  • 空間複雜度：O(1)O(1)O(1)。

方法三：雜湊表(Hash Table)

  • 使用雜湊表來記錄陣列中的所有數，然後遍歷 [0,n][0, n][0,n] ，找到缺失的數。
  • 時間複雜度：O(n)O(n)O(n)。
  • 空間複雜度：O(n)O(n)O(n)。

方法四：位運算(Bit Manipulation)

  • 利用 XORXORXOR 運算的性質，即 a⊕a=0a \oplus a = 0a⊕a=0 且 a⊕0=aa \oplus 0 = aa⊕0=a 。
  • 將 [0,n][0, n][0,n] 和陣列中的所有數進行 XOR 運算，最後的結果即為缺失的數。
  • 時間複雜度：O(n)O(n)O(n)。
  • 空間複雜度：O(1)O(1)O(1)。
