題意

  • 給定兩個整樹 leftleftleft 和 rightrightright ，表示區間 [left,right][left, right][left,right] ，返回此區間內所有數字 按位與(bitwise AND) 的結果（包含 leftleftleft 、rightrightright ）。

思路：位運算(Bit Manipulation)

  • 空間複雜度：O(1)O(1)O(1)。

方法二：Brian Kernighan Algorithm

  • 透過 x&(x−1)x \& (x-1)x&(x−1) 可以消除 xxx 的最低位的 111 ，所以我們可以逐位消除 rightrightright 的最低位的 111 ，直到 right<=leftright <= leftright<=left 。

🟡 787. Cheapest Flights Within K Stops 1786

題意

  • 有 nnn 個城市透過一些航班連接，給定一個陣列 flightsflightsflights ，其中 flights[i]=[fromi,toi,pricei]flights[i] = [from_i, to_i, price_i]flights[i]=[fromi​,toi​,pricei​] ，表示該航班都從城市 fromifrom_ifromi​ 開始，以價格 priceiprice_ipricei​ 抵達 toito_itoi​。

經過 kkk 個中轉站，為從 srcsrcsrc 開始 走 k+1k+1k+1 步 (包含終點)

方法一：BFS + Pruning

  • 由於要在 k+1k+1k+1 步內找到最低價格，我們可以使用 BFS 來進行搜尋。
  • 將 (u,d,s)(u, d, s)(u,d,s) 加入 BFS Queue 中，其中 uuu 是當前城市，ddd 是從 srcsrcsrc 到 uuu 的距離，sss 是經過的中轉站數。

  • 若以BFS遍歷完所有圖，則很有可能會超時，因此我們可以進行 剪枝(Pruning) ，當 s>ks > ks>k 時，則不再進行BFS。
  • 時間複雜度：O(k⋅n2)O(k \cdot n^2)O(k⋅n2)，其中 nnn 為城市數，最多遍歷 kkk 層，每層最多有 n−1n-1n−1 個元素，每個元素最多有 n−1n-1n−1 個相鄰節點。

  • 令 dp[i][k]dp[i][k]dp[i][k] 表示在 kkk 步內，從 srcsrcsrc 到 iii 的最小花費。

  • 空間複雜度：O(n⋅k)O(n \cdot k)O(n⋅k) 或 O(n)O(n)O(n)。

  • Bellman-Ford 演算法是一種 單源最短路算法 ，可以處理有負權邊的圖。
  • 每次迭代，我們都對所有的航班進行 鬆弛(relaxation) 操作，即 dist[v]=min⁡(dist[v],dist[u]+w)dist[v] = \min(dist[v], dist[u] + w)dist[v]=min(dist[v],dist[u]+w) ，經過 k+1k+1k+1 次迭代後，可得到從 srcsrcsrc 到 vvv 經過 k+1k+1k+1 步 (即 kkk 次中轉)的最小花費，最後返回 dist[dst]dist[dst]dist[dst] 即為答案。

  • 由於 Bellman-Ford 的核心是進行 k+1k+1k+1 次迭代，每次迭代都對所有的航班進行 鬆弛(relaxation) 操作，因此我們可以不需要建立圖，直接使用 flightsflightsflights 進行鬆弛。

  • 專家 000 知道一個 秘密 。最初，他在時間 000 將這個秘密分享給了專家 firstPersonfirstPersonfirstPerson 。 接著，這個秘密會在每次有知道這個秘密的專家參加會議時傳遞。在每次會議中，如果專家 xix_ixi​ 在時間 timeitime_itimei​ 時知道這個秘密，那麼他將會與專家 yiy_iyi​ 分享這個秘密，反之亦然。
  • 秘密傳遞是 瞬間發生 的，也就是說，在同一時間，一個專家不光可以接收到秘密，還能在其他會議上與其他專家分享。
  • 在所有會議都結束之後，返回所有知曉這個秘密的專家清單，可以按照 任何順序 返回答案。

思路：Disjoint Set Union(DSU)

  • 這題的關鍵在於如何模擬秘密的傳遞，知道秘密和不知道秘密的專家可以看作是不同集合(Set)，而進行會議則是對這兩個集合進行 聯集(Union) 操作，因此可以使用 Disjoint Set Union (DSU) 來模擬秘密的傳遞。
  • 首先，我們可以依照專家的編號建立一個大小為 nnn 的 Disjoint Set Union(DSU) ，並且將每個專家的父節點初始化為自己，即將每個專家視為一個集合。
  • 依照題意，在初始化時，專家 000 知道秘密，且告知了專家 firstPersonfirstPersonfirstPerson ，因此我們可以將專家 000 與 firstPersonfirstPersonfirstPerson 進行 聯集(Union) 操作。且因為會議進行需按照時間先後順序，我們可以將會議按照時間進行排序。

題意

  • 給定一棵二元樹的根節點 rootrootroot，返回 最後一層中最左邊 節點的值。
  • 假設至少有一個節點。

思路：BFS(Level Order Traversal)

方法一：每次保存該層的所有節點

  • 按照題義，使用BFS遍歷每一層的節點，並在遍歷的同時保存該層的所有節點，最後返回最後一層的第一個節點的值。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)。

方法二：由右至左層序遍歷

  • 由於題目要求返回最後一層中最左邊節點的值，因此我們可以使用由右至左的層序遍歷(Level Order Traversal)，即每次先將右子節點加入佇列(queue)，再將左子節點加入佇列，最後返回最後一個節點的值即可。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)，但不用另外保存每一層的所有節點。

🔴 2581. Count Number of Possible Root Nodes 2228

一開始用記憶化搜索，雖然能AC，但發現執行時間很差，看了題解後才知道可以用換根DP，效率提升很多。

題意

  • Alice有一棵 nnn 個節點的樹，節點編號為 000 到 n−1n - 1n−1。給定一個長度為 n−1n - 1n−1 的二維整數陣列 edgesedgesedges，其中 edges[i]=[ai,bi]edges[i] = [a_i, b_i]edges[i]=[ai​,bi​] 表示樹中節點 aia_iai​ 和 bib_ibi​ 之間有一條邊。

    • 選擇兩個 不相等 的整數 uuu 和 vvv，且樹中必須存在邊 [u,v][u, v][u,v]。
    • Bob猜測樹中 uuu 是 vvv 的 父節點 。
  • Bob的猜測用二維整數陣列 guessesguessesguesses 表示，其中 guesses[j]=[uj,vj]guesses[j] = [u_j, v_j]guesses[j]=[uj​,vj​] 表示Bob猜 uju_juj​ 是 vjv_jvj​ 的父節點。
  • Alice 只會告訴Bob這些猜測裡面 至少 有 kkk 個猜測的結果為 truetruetrue 。
  • 給定二維整數陣列 edgesedgesedges，Bob的所有猜測 guessesguessesguesses 和整數 kkk，請你返回可能成為樹根的 節點數目 。如果沒有這樣的樹，則返回 000。

思路：DFS + 記憶化搜索(Memoization) DP / 換根(Rerooting) DP

方法一：DFS + 記憶化搜索(Memoization) DP

  • 由於給定了樹中的所有邊，因此可以假設某個節點為根，並進行深度優先搜索(DFS)來計算以該節點為根時的猜對數量。
  • 由於可能會有多次DFS，因此可以使用 記憶化搜索(Memoization) 來避免重複計算。
  • 時間複雜度：O(n2)O(n^2)O(n2)，其中 nnn 為節點數。
  • 空間複雜度：O(n2+n)O(n^2 + n)O(n2+n)，建圖需要 O(n2)O(n^2)O(n2) 的空間，記憶化搜索需要 O(n)O(n)O(n) 的空間。
    • 對於每個節點 uuu，以及相鄰的節點 v,wv, wv,w，需要紀錄 uuu 為樹根，uuu 的父節點為 vvv，uuu 的父節點為 www，總共 333 種狀態。

方法二：DFS + 換根(Rerooting) DP

  • 由於給定了樹中的所有邊，因此可以假設某個節點為根，並進行深度優先搜索(DFS)來計算以該節點為根時的猜對數量。
  • 但每次換根時，只會改變 111 組邊的父子關係，因此可以使用換根(Rerooting) DP來計算答案。

    如果 xxx 和 yyy 相鄰，那麼從「以 xxx 為根的樹」變成「以 yyy 為根的樹」時，只有 xxx 和 yyy 的父子關係改變，其餘相鄰節點的父子關係沒有變化。所以只有 [x,y][x,y][x,y] 和 [y,x][y,x][y,x] 這兩個猜測的正確性變了，其餘猜測的正確性不變。 (來自參考資料)

  • 因此可以先計算出以 000 為根時的猜對數量後再進行一次DFS，這次DFS的目的是計算換根後的猜對數量，若 yyy 是 xxx 的子節點，則 yyy 的猜對數量的變化為 cnt−((x,y)inst)+((y,x)inst)cnt - ((x, y) in st) + ((y, x) in st)cnt−((x,y)inst)+((y,x)inst)。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為節點數。
  • 空間複雜度：O(n2+n)O(n^2 + n)O(n2+n)，建圖需要 O(n2)O(n^2)O(n2) 的空間，換根DP需要 O(n)O(n)O(n) 的空間。

🟡 1609. Even Odd Tree 1438

題意

  • 若一個二元樹(Binary Tree)滿足以下幾個條件，則可以稱為 奇偶二元樹(Even Odd Tree) ：
    1. 設二元樹的根節點所在層下標為 000，根的子節點所在層下標為 111，根的孫節點所在層下標為 222，依此類推。
    2. 對於每個下標為偶數的偶數層，該層上的所有節點都為 嚴格遞增 的 奇整數（從左到右）。
    3. 對於每個下標為奇數的奇數層，該層上的所有節點都為 嚴格遞減 的 偶整數（從左到右）。
  • 給定一個二元樹的根節點 rootrootroot，如果該二元樹是 奇偶二元樹 ，則返回 truetruetrue；否則返回 falsefalsefalse。

思路：BFS / DFS

方法一：BFS(Level Order Traversal)保存每一層的節點值

  • 使用廣度優先搜索(BFS)遍歷每個節點，並使用一個陣列 arrarrarr 來保存每一層的節點值，再依據該層的下標來判斷是否滿足奇偶性以及遞增或遞減的條件即可。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)。

方法二：BFS(Level Order Traversal)，不保存每一層的節點值

  • 在方法一的基礎上優化，不保存每一層的節點值，而是直接在BFS遍歷的過程中判斷是否滿足奇偶性以及遞增或遞減的條件即可。
  • 為滿足在遍歷的同時檢查是否滿足條件，可以使用一個變數 prevprevprev 來保存上一個節點的值，在每層開始時根據 levellevellevel 將 prevprevprev 初始化為 −INF-\text{INF}−INF (偶數層遞增) 或 +INF+\text{INF}+INF (奇數層遞減)，並在遍歷每個節點時檢查是否滿足條件。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)。

方法三：DFS + Hash Table

  • 根據和方法二同樣的思路，我們也可以使用深度優先搜索(DFS)遍歷每個節點，但由於不會在遍歷完該層所有節點後才進入下層，因此需要使用一個Hash Table lastlastlast 保存每層最後一個節點的值。
  • 在訪問每個節點時，我們可以根據該節點的值和 lastlastlast 來判斷是否滿足條件。
  • 時間複雜度：O(n)O(n)O(n)，其中 nnn 為樹的節點數。
  • 空間複雜度：O(n)O(n)O(n)。
