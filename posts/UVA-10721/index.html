🔗 UVA-10721 Bar Codes

tags: 動態規劃(DP), 記憶化搜尋(Memoization)

範例程式碼已於UVA、瘋狂程設(CPE) 上皆測試通過 (此題無 ZeroJudge )。

題意

條碼(bar code) 是由黑色和白色的 單元(unit) 組成，且由黑色的單元開始，若干個 unit 可以組成一個 長條(bar) ，而若干個 bar 又能組成一個 bar code 。

給定三個整數 n,k,mn, k, mn,k,m，代表條碼的長度為 nnn 個 unit 、具有 kkk 個 bar 、每個 bar 的寬度最多為 mmm 個 unit ，求有多少種不同的條碼可以滿足條件，即求 BC(n,k,m)BC(n, k, m)BC(n,k,m)。

思路：動態規劃(DP)

來自 @灵茶山艾府

求解 BC(n,k,m)BC(n, k, m)BC(n,k,m) 時，我們可以考慮第一個 bar 的寬度 xxx，將問題分解成求解 BC(n−x,k−1,m)BC(n-x, k-1, m)BC(n−x,k−1,m) 之子問題，也就是剩下 n−xn-xn−x 個 units ，k−1k-1k−1 個 bars ，每個 bars 最多 mmm 個 units 時的答案。將這些子問題的答案加總即為 BC(n,k,m)BC(n, k, m)BC(n,k,m) 的答案。

再來考慮 xxx 的範圍，顯然 xxx 不能超過目前可用的 unit 數量 nnn，且每個 bar 的寬度最多為 mmm 個 units ，因此 xxx 的範圍為 [1,min⁡(n,m)][1, \min(n, m)][1,min(n,m)]。

由於每次的顏色必須和上次不同，且開頭也只能是黑色，因此在考慮每個子問題時，都只需當作只有一種顏色即可。

最後來寫邊界條件：

  1. 對於 n=0n=0n=0 時，若 k=0k=0k=0 代表能夠滿足條件，返回 111；否則代表 bar 的數量不夠，返回 000。
  2. 對於 k=1k=1k=1 時，若 n≤mn \leq mn≤m 代表能夠滿足條件，返回 111；否則代表 bar 的寬度超過限制，返回 000。

複雜度分析

  • 時間複雜度：O(N3)O(N^3)O(N3)，其中 NNN 為 nnn 的最大值，對於每個詢問。
    • 由於在同一個詢問中 mmm 值固定，因此總共有 O(N2)O(N^2)O(N2) 個子問題，求解每個子問題的時間複雜度為 O(N)O(N)O(N)。
    • 由於 k,m≤nk, m \leq nk,m≤n ，故考慮 nnn 的最大值即可。
  • 空間複雜度：O(N3)O(N^3)O(N3)，用於存儲子問題的答案。
  • 使用 Top-Down 的方式實作動態規劃，並使用 functools.cache 進行 記憶化搜尋(Memoization) 。
