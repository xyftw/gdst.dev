🔗 B - Heavy Snake (abc388 B)

tags: 模擬(Simulation)

題意

有 NNN 條蛇。一開始，第 iii 條蛇的粗細為 TiT_iTi​，長度為 LiL_iLi​。蛇的重量定義為其粗細 TiT_iTi​ 與長度 LiL_iLi​ 的乘積。

現在你可以增加每條蛇的長度，增加的長度為 kkk，其中 kkk 為 111 到 DDD 之間的整數。對於每個滿足 1≤k≤D1 \leq k \leq D1≤k≤D 的整數 kkk，找出當每條蛇的長度都增加 kkk 後，最重的蛇的重量。

約束條件

  • 1≤N,D≤1001 \leq N, D \leq 1001≤N,D≤100

思路：模擬(Simulation)

注意到 DDD 的範圍很小，因此可以對 kkk 進行枚舉，分別計算每個 kkk 時，最重的蛇的重量。

複雜度分析

  • 時間複雜度：O(D×N)O(D \times N)O(D×N)
  • 空間複雜度：O(N)O(N)O(N)

🔗 C - Various Kagamimochi (abc388 C)

tags: 雙指標(Two Pointers) 枚舉右維護左

題意

有 NNN 個年糕，第 iii 個年糕的大小為 AiA_iAi​，其中 (1≤i≤N)(1 \leq i \leq N)(1≤i≤N)，並且 AiA_iAi​ 已經按照大小以升序排列。

製作一個鏡餅（疊加的年糕）需要將兩個年糕疊放在一起，且上面年糕的大小不超過下面年糕的一半。具體來說，給定兩個年糕 AAA 和 BBB，其大小分別為 aaa 和 bbb，當且僅當 a≤b2a \leq \frac{b}{2}a≤2b​ 時，可以通過將年糕 AAA 放在年糕 BBB 上來製作一個鏡餅。

找出在給定的 NNN 個年糕中，可以製作多少種不同的鏡餅。如果至少有一個年糕不同，即使年糕的大小相同，兩個鏡餅也是不同的。

約束條件

  • 2≤N≤5×1052 \leq N \leq 5 \times 10^52≤N≤5×105
  • 1≤Ai≤109 (1≤i≤N)1 \leq A_i \leq 10^9 \ (1 \leq i \leq N)1≤Ai​≤109 (1≤i≤N)
  • Ai≤Ai+1 (1≤i<N)A_i \leq A_{i+1} \ (1 \leq i < N)Ai​≤Ai+1​ (1≤i<N)

思路：枚舉右維護左

由於 AiA_iAi​ 已經按照大小以升序排列，且當增加 bbb 的大小時，滿足條件的 aaa 的數量只會增加，不會減少。

因此可以使用 雙指標(Two Pointers) 枚舉右端點，並維護能疊在右端點上的左端點數量。

複雜度分析

  • 時間複雜度：O(N)O(N)O(N)
  • 空間複雜度：O(N)O(N)O(N)

🔗 D - Coming of Age Celebration (abc388 D)

tags: 差分陣列(Difference Arrays) 前綴和(Prefix Sum)

題意

在某個星球上，有 NNN 位外星人，且他們都是未成年人。第 iii 位外星人目前擁有 AiA_iAi​ 顆石頭，並將在正好 iii 年後成為成年人。

當某人在這個星球上成為成年人時，每位至少擁有一顆石頭的 成年人 都會贈送恰好一顆石頭作為賀禮給剛成年的外星人。

找出在 NNN 年後每位外星人將擁有多少顆石頭。假設未來不會有新的外星人誕生。

約束條件

  • 1≤N≤5×1051 \leq N \leq 5 \times 10^51≤N≤5×105
  • 0≤Ai≤5×1050 \leq A_i \leq 5 \times 10^50≤Ai​≤5×105

思路：差分陣列(Difference Arrays)

首先注意到題目中的第 iii 位外星人將在 iii 年後成年，因此成年的順序即為給定的陣列順序 1,2,⋯ ,N1, 2, \cdots, N1,2,⋯,N。接著，將 iii 從 jjj 收到石頭視為 jjj 給出石頭給 iii，其中 j<ij < ij<i。如此便能在 iii 成年時計算其收到的石頭數量，並依此計算給出的石頭數量。

轉換後便可以從維護收到的石頭數量下手，對於第 iii 位外星人，若其能夠給出 ggg 個石頭，則編號為 [i+1,i+g][i+1, i+g][i+1,i+g] 的外星人，其收到石頭數量增加 111，這種區間加操作可以用 差分陣列(Difference Arrays) 維護。

對差分陣列做前綴和即可得到 iii 從前面的人那裡收到的石頭數量 curcurcur。而根據原本的石頭數量 AiA_iAi​ 和 curcurcur，以及後面的人數量 N−iN - iN−i，可以算出給出的石頭數量 ggg，最終第 iii 位外星人的石頭數量為 Ai+cur−gA_i + cur - gAi​+cur−g。

複雜度分析

  • 時間複雜度：O(N)O(N)O(N)
  • 空間複雜度：O(N)O(N)O(N)

🔗 E - Simultaneous Kagamimochi (abc388 E)

tags: 二分搜尋(Binary Search) 貪心(Greedy) 雙指標(Two Pointers) 預處理(Preprocessing)

題意

有 NNN 個年糕，第 iii 個年糕的大小為 AiA_iAi​，其中 (1≤i≤N)(1 \leq i \leq N)(1≤i≤N)，並且 AiA_iAi​ 已經按照大小以升序排列。

製作一個鏡餅（疊加的年糕）需要將兩個年糕疊放在一起，且上面年糕的大小不超過下面年糕的一半。具體來說，給定兩個年糕 AAA 和 BBB，其大小分別為 aaa 和 bbb，當且僅當 a≤b2a \leq \frac{b}{2}a≤2b​ 時，可以通過將年糕 AAA 放在年糕 BBB 上來製作一個鏡餅。

現在你需要計算可以同時製作多少個鏡餅。

更精確地說，找出最大的非負整數 KKK，使得以下條件成立：

  • 從 NNN 個年糕中選出 2K2K2K 個，形成 KKK 對。對於每一對，將其中一個年糕疊在另一個上，製作 KKK 個鏡餅。

約束條件

  • 2≤N≤5×1052 \leq N \leq 5 \times 10^52≤N≤5×105
  • 1≤Ai≤109 (1≤i≤N)1 \leq A_i \leq 10^9 \ (1 \leq i \leq N)1≤Ai​≤109 (1≤i≤N)
  • Ai≤Ai+1 (1≤i<N)A_i \leq A_{i+1} \ (1 \leq i \lt N)Ai​≤Ai+1​ (1≤i<N)

思路：貪心(Greedy) + 二分搜尋(Binary Search)

基於貪心原則，為了盡可能組成 kkk 對，則上面的 kkk 個年糕應盡可能的小，也就是說，在上面的 kkk 個年糕最好是最小的 kkk 個。

方法一：O(Nlog⁡N)O(N \log N)O(NlogN)

而若存在 kkk 對，則最小的 kkk 個年糕和區間內其他 kkk 個年糕存在有效的配對，此時將某個年糕與尚未使用且更大的年糕交換，必也能配對成功。換句話說，若存在 kkk 對，則區間內最小的 kkk 個年糕和最大 kkk 個年糕必存在有效的配對。因此，判斷是否存在 kkk 對，只需要檢查這 2k2k2k 個年糕是否能配對成功即可，故檢查函數 check() 的時間複雜度為 O(k)O(k)O(k)。

在得到檢查函數後，由於 check() 函數有越小越合法的單調性，因此可以對 kkk 進行二分搜尋，由於最大的配對數量為 ⌊N2⌋\lfloor \frac{N}{2} \rfloor⌊2N​⌋，因此需要搜尋 O(log⁡N)O(\log N)O(logN) 次。

複雜度分析

  • 時間複雜度：O(Nlog⁡N)O(N \log N)O(NlogN)。總共進行二分搜尋 O(log⁡N)O(\log N)O(logN) 次，每次檢查需要 O(k)O(k)O(k) 時間，其中 kkk 為配對數量，可以視為 O(N)O(N)O(N)。
  • 空間複雜度：O(N)O(N)O(N)

方法二： O(N)O(N)O(N)

這個思路可以用在 G 題，以下用 [L,R][L, R][L,R] 表示搜尋區間，本題中只需考慮 L=1,R=NL = 1, R = NL=1,R=N 的情況。

若區間 [L,R][L, R][L,R] 內可以組成 KKK 對，則最小的 KKK 個數和最大的 KKK 個數必能配對成功，其中最小的 KKK 個數為 [L,L+K−1][L, L + K - 1][L,L+K−1]，最大的 KKK 個數為 [R−K+1,R][R - K + 1, R][R−K+1,R]。

但若最小的 KKK 個數中有一個數無法配對成功，則會影響到後續的數，使後續的數需要 往後遞移 ，因此其實只考慮最小的 KKK 個數的右端點 L+K−1L + K - 1L+K−1 、以及 [L,L+K−1][L, L + K - 1][L,L+K−1] 中的最大間距 max_gapmax\_gapmax_gap 即可。

  • 如果 L+K−1+max_gap>RL + K - 1 + max\_gap > RL+K−1+max_gap>R，則說明往後遞移之後會超過範圍，也就是說 KKK 太大了，需要縮小。

根據此思路，可以使用雙指標預處理出每個位置與下一個符合條件的位置的間距，由於本題中左端點固定為 111，因此可以維護前綴最大間距即可，如此便可以在 O(1)O(1)O(1) 時間內查詢區間內的最大間距。

複雜度分析

  • 時間複雜度：O(N+log⁡N)=O(N)O(N + \log N) = O(N)O(N+logN)=O(N)
    • 預處理每個位置與下一個符合條件的位置的間距、計算前綴最大間距都需要 O(N)O(N)O(N) 時間。
    • 二分搜尋需要 O(log⁡N)O(\log N)O(logN) 次，在預處理後，每次檢查只需要 O(1)O(1)O(1) 時間。
  • 空間複雜度：O(N)O(N)O(N)

🔗 G - Simultaneous Kagamimochi 2 (abc388 G)

tags: 貪心(Greedy) 二分搜尋(Binary Search) 線段樹(Segment Tree)

題意

有 NNN 個年糕，第 iii 個年糕的大小為 AiA_iAi​，其中 (1≤i≤N)(1 \leq i \leq N)(1≤i≤N)，並且 AiA_iAi​ 已經按照大小以升序排列。

製作一個鏡餅（疊加的年糕）需要將兩個年糕疊放在一起，且上面年糕的大小不超過下面年糕的一半。具體來說，給定兩個年糕 AAA 和 BBB，其大小分別為 aaa 和 bbb，當且僅當 a≤b2a \leq \frac{b}{2}a≤2b​ 時，可以通過將年糕 AAA 放在年糕 BBB 上來製作一個鏡餅。

現在給定 QQQ 區間，令 (Li,Ri)(L_i , R_i)(Li​,Ri​) 為第 iii 個區間 (1≤i≤Q)(1\leq i\leq Q)(1≤i≤Q)，並為每個 iii 解決以下問題：

  • 僅使用從第 [Li,Ri][L_i, R_i][Li​,Ri​] 之間的 Ri−Li+1R_i - L_i + 1Ri​−Li​+1 個麻糬，最多可以同時製作多少個鏡餅？
  • 更精確地說，找出最大的非負整數 KKK，使得以下條件成立：
    • 從第 [Li,Ri][L_i, R_i][Li​,Ri​] 之間的 Ri−Li+1R_i - L_i + 1Ri​−Li​+1 個麻糬中，選出 2K2K2K 個麻糬並形成 KKK 對。對於每一對，將一個麻糬放在另一個上面，製作 KKK 個鏡餅。

約束條件

  • 2≤N≤2×1052 \leq N \leq 2 \times 10^52≤N≤2×105
  • 1≤Ai≤109 (1≤i≤N)1 \leq A_i \leq 10^9 \ (1 \leq i \leq N)1≤Ai​≤109 (1≤i≤N)
  • Ai≤Ai+1 (1≤i<N)A_i \leq A_{i+1} \ (1 \leq i < N)Ai​≤Ai+1​ (1≤i<N)
  • 1≤Q≤2×1051 \leq Q \leq 2 \times 10^51≤Q≤2×105
  • 1≤Li<Ri≤N (1≤i≤Q)1 \leq L_i < R_i \leq N \ (1 \leq i \leq Q)1≤Li​<Ri​≤N (1≤i≤Q)

思路：貪心 + 二分搜尋 + 線段樹

本題和 EEE 題類似，都需要求區間內的最大配對數，但如果使用 O(k)O(k)O(k) 的檢查函數做二分搜尋會超時，因此需要和方法二一樣預處理後，利用該性質進行二分搜尋。

若區間 [L,R][L, R][L,R] 內可以組成 KKK 對，則最小的 KKK 個數和最大的 KKK 個數必能配對成功，其中最小的 KKK 個數為 [L,L+K−1][L, L + K - 1][L,L+K−1]，最大的 KKK 個數為 [R−K+1,R][R - K + 1, R][R−K+1,R]。

但若最小的 KKK 個數中有一個數無法配對成功，則會影響到後續的數，使後續的數需要 往後遞移 ，因此其實只考慮最小的 KKK 個數的右端點 L+K−1L + K - 1L+K−1 、以及 [L,L+K−1][L, L + K - 1][L,L+K−1] 中的最大間距 max_gapmax\_gapmax_gap 即可。

  • 如果 L+K−1+max_gap>RL + K - 1 + max\_gap > RL+K−1+max_gap>R，則說明往後遞移之後會超過範圍，也就是說 KKK 太大了，需要縮小。

根據此思路，可以預先計算出每個位置與下一個符合條件的位置的間距，並使用 線段樹(Segment Tree) 維護區間內的最大間距，如此便可以在 O(log⁡N)O(\log N)O(logN) 時間內查詢區間 [L,L+K−1][L, L + K - 1][L,L+K−1] 內的最大間距，檢查函數為 L+K−1+max_gap≤RL + K - 1 + max\_gap \leq RL+K−1+max_gap≤R。

如果對線段樹不夠熟練，或是使用的線段樹模板不夠好的話，可能會超時，因此這裡直接使用 Atcoder Library 的線段樹模板。

複雜度分析

  • 時間複雜度：O(N+Q⋅(log⁡N)2)O(N + Q \cdot (\log N)^2)O(N+Q⋅(logN)2)
    • 預處理每個位置與下一個符合條件的位置的間距，並以此建立線段樹需要 O(N)O(N)O(N) 時間。
    • 二分搜尋需要 O(log⁡N)O(\log N)O(logN) 次，在預處理後，每次檢查只需要查詢線段樹，時間為 O(log⁡N)O(\log N)O(logN)，故對於每個詢問，時間複雜度為 O((log⁡N)2)O((\log N)^2)O((logN)2)。
  • 空間複雜度：O(N)O(N)O(N)
