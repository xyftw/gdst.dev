🔗 945. Minimum Increment to Make Array Unique 1448

tags: Weekly Contest 112 貪心(Greedy) 排序(Sorting) 計數(Counting)

題意

給定一個整數陣列 numsnumsnums 。

在每次操作中，可以選擇一個滿足 0≤i<nums.length0 \leq i < \text{nums.length}0≤i<nums.length 的索引 iii，並將 nums[i]\text{nums}[i]nums[i] 增加 111。

返回使 nums\text{nums}nums 中每個值都 不同 所需的最小操作次數。

約束條件

  • 1≤nums.length≤1051 \leq \text{nums.length} \leq 10^51≤nums.length≤105
  • 0≤nums[i]≤1050 \leq \text{nums[i]} \leq 10^50≤nums[i]≤105
  • 測試案例產生的答案皆在 323232 位元整數的範圍內。

思路

方法一：貪心(Greedy) + 排序(Sorting)

首先來看一個比較直覺的方法，由於在每次操作中都只能使某個位置變大，且所求的答案和原本陣列中的元素位置無關。因此可以將陣列由小到大排序後，從左到右遍歷，對於每個位置 iii，如果 nums[i]≤nums[i−1]\text{nums}[i] \leq \text{nums}[i-1]nums[i]≤nums[i−1]，則為了使陣列中的元素不重複，需要將 nums[i]\text{nums}[i]nums[i] 調整到 nums[i−1]+1\text{nums}[i-1] + 1nums[i−1]+1，並將增加的次數加到答案中。

複雜度分析

  • 時間複雜度：O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) ，排序需要 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) 的時間，遍歷排序後的陣列需要 O(n)\mathcal{O}(n)O(n) 的時間。
  • 空間複雜度：O(1)\mathcal{O}(1)O(1)，忽略排序所需的空間。

方法二：計數(Counting)

注意到約束條件中的 nums[i]≤105\text{nums[i]} \leq 10^5nums[i]≤105，且 nums.length≤105\text{nums.length} \leq 10^5nums.length≤105，因此在最差情況下，操作後的元素值可能會達到 105+105=2×10510^5 + 10^5 = 2 \times 10^5105+105=2×105，因此可以使用基於 值域 的計數方法。

維護一個長度為 2×1052 \times 10^52×105 的陣列 cntcntcnt，其中 cnt[i]cnt[i]cnt[i] 表示數字 iii 出現的次數。遍歷值域中的每個數字 xxx ，xxx 出現次數大於 111，則代表有 cnt[x]−1cnt[x] - 1cnt[x]−1 個 xxx 需要被調整成更大的數字，因此先把這些多餘的數字加到一個陣列 leftleftleft 中。當遇到下一個數字時，如果該數字沒有出現過，則可以從 leftleftleft 中取出一個多餘的數字，將其調整到當前數字，並將答案加上需要的操作次數。

雖然直接使用長度為 2×1052 \times 10^52×105 的陣列也能通過，但是為了節省空間，可以先找出最大值 mxmxmx，由於最差情況下為 numsnumsnums 中的所有元素皆為 mxmxmx，因此操作完成後的值域為 [0,mx+n)[0, mx + n)[0,mx+n)，故使用一個長度為 mx+nmx + nmx+n 的陣列即可。

複雜度分析

  • 時間複雜度：O(mx+n)\mathcal{O}(mx + n)O(mx+n)，其中 nnn 是陣列 numsnumsnums 的長度，mxmxmx 是陣列中的最大值。
    • 遍歷陣列 numsnumsnums 需要 O(n)\mathcal{O}(n)O(n) 的時間。
    • 遍歷 cntcntcnt 陣列需要 O(mx+n)\mathcal{O}(mx + n)O(mx+n) 的時間。
  • 空間複雜度：O(mx+n)\mathcal{O}(mx + n)O(mx+n)，使用了長度為 mx+nmx + nmx+n 的陣列 cntcntcnt ，以及長度最大為 O(n)O(n)O(n) 的陣列 leftleftleft。

方法二優化

在方法二中，我們需要使用一個陣列 leftleftleft 來存放多餘的數字，使得在每次遇到新的數字時，需要從 leftleftleft 中取出一個數字。這裡可以進一步優化，不需要使用 leftleftleft 陣列，只需要維護一個整數 left_cntleft\_cntleft_cnt 來表示多餘的數字個數即可。

由於枚舉到沒有出現過的數字 xxx 時，需要從 leftleftleft 中取出一個數字 yyy，將 yyy 調整到 xxx，此時需要的操作次數為 x−yx - yx−y 。可以看出若 cnt[y]≥2cnt[y] \geq 2cnt[y]≥2 ，則其對答案的貢獻為 −y×(cnt[y]−1)-y \times (cnt[y] - 1)−y×(cnt[y]−1)，因此可以提前計算 yyy 的貢獻，並將其加到答案中。如此便不用維護 leftleftleft 陣列。

但在枚舉值域中的數字時，仍然需要紀錄前面是否出現過多餘的數字，因此需要維護一個整數 left_cntleft\_cntleft_cnt 來表示多餘的數字個數。

複雜度分析

  • 時間複雜度：O(mx+n)\mathcal{O}(mx + n)O(mx+n)，其中 nnn 是陣列 numsnumsnums 的長度，mxmxmx 是陣列中的最大值。
    • 遍歷陣列 numsnumsnums 需要 O(n)\mathcal{O}(n)O(n) 的時間。
    • 遍歷 cntcntcnt 陣列需要 O(mx+n)\mathcal{O}(mx + n)O(mx+n) 的時間。
  • 空間複雜度：O(mx+n)\mathcal{O}(mx + n)O(mx+n)，使用了長度為 mx+nmx + nmx+n 的陣列 cntcntcnt。
