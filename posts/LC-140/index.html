🔗 🔴 140. Word Break II

tags: 動態規劃(Dynamic Programming) 記憶化搜尋(Memoization) 回溯(Backtracking) 剪枝(Pruning) 字典樹(Trie) 子集型回溯 位運算(Bit Manipulation) 狀態壓縮 雜湊表(Hash Table)

題意

給定一個字串 sss 和一個字串陣列 wordDictwordDictwordDict，在字串 sss 中加入空格來構建一個句子，使得句子中所有的單字都在 wordDictwordDictwordDict 中。以 任意順序 返回所有這樣的可能句子。

注意，句子中可能會包含多個相同的單字 。

限制

  • 1≤s.length≤201 \leq s.length \leq 201≤s.length≤20
  • 1≤wordDict.length≤10001 \leq wordDict.length \leq 10001≤wordDict.length≤1000
  • 1≤wordDict[i].length≤101 \leq wordDict[i].length \leq 101≤wordDict[i].length≤10
  • sss 和 wordDict[i]wordDict[i]wordDict[i] 皆由小寫英文字母組成

思路

方法一：動態規劃(Dynamic Programming) + 記憶化搜尋(Memoization) + 雜湊表(Hash Table)

類似於 139. Word Break 的動態規劃解法，同樣可以利用劃分型DP來解決，但在本題中需要返回所有可能的劃分結果。

因此令 dp[i]dp[i]dp[i] 表示 s[i:]s[i:]s[i:] 的所有可能劃分，則可以枚舉第一個單字的終點 jjj ，若 s[i:j+1]s[i:j+1]s[i:j+1] 在 wordDictwordDictwordDict 中，則 s[i:j+1]s[i:j+1]s[i:j+1] 可以作為劃分的第一個單字，而 s[j+1:]s[j+1:]s[j+1:] 的所有可能劃分即為 dp[j+1]dp[j+1]dp[j+1]，因此 s[i:]s[i:]s[i:] 的所有可能劃分即為 s[i:j+1]s[i:j+1]s[i:j+1] 和 dp[j+1]dp[j+1]dp[j+1] 的所有可能劃分的組合。

而在計算過程中會出現許多相同的子問題，因此可以使用 記憶化搜尋(Memoization) 來避免重複計算，在 Python 中使用 @cache 裝飾器即可。

此外，由於 Python 中可以使用 join() 方法將列表轉換為字串，因此 Python 的範例程式碼中，是先保存所有可能劃分的單字列表，最後再使用 join() 方法將其轉換為字串；在 C++ 的範例程式碼中則是直接將所有可能劃分的單字串接起來。

為了更快檢查 wordwordword 是否在 wordDictwordDictwordDict 中，可以先將陣列 wordDictwordDictwordDict 轉換為集合 wordSetwordSetwordSet。

複雜度分析

  • 時間複雜度 O(n⋅2n)O(n \cdot 2^n)O(n⋅2n)，由於需要將子問題的結果添加到結果中，因此時間複雜度為 O(n⋅2n)O(n \cdot 2^n)O(n⋅2n)
  • 空間複雜度 O(n⋅2n)O(n \cdot 2^n)O(n⋅2n)。
    • 最差情況下任何一個點都可以劃分，因此共有 2n−12^{n-1}2n−1 種劃分方式，每種劃分方式的長度為 O(n)O(n)O(n)，因此空間複雜度為 O(n⋅2n)O(n \cdot 2^n)O(n⋅2n) 。
    • 就算答案所使用的空間可以被忽略，但因為使用了記憶化搜索，計算過程中的空間複雜度仍然為 O(n⋅2n)O(n \cdot 2^n)O(n⋅2n)

方法二：回溯(Backtracking) + 字典樹(Trie)

在方法一中，需要枚舉所有的終點 jjj ，才能檢查 word=s[i:j+1]word = s[i:j+1]word=s[i:j+1] 是否在 wordDictwordDictwordDict 中，那有沒有一個辦法可以在檢查到 jjj 時，就可以確定 j+1j+1j+1 也不會是劃分的終點呢？

為了達到這個目的，可以使用 字典樹(Trie) 來保存 wordDictwordDictwordDict 中的所有單字，這樣在檢查 s[i:j+1]s[i:j+1]s[i:j+1] 是否在 wordDictwordDictwordDict 中時，只需要在 Trie 中查找即可。

  • 若 s[i:j+1]s[i:j+1]s[i:j+1] 代表的節點不在 Trie 中，則 s[i:j+2]s[i:j+2]s[i:j+2] 一定不會在 wordDictwordDictwordDict 中。
  • 若 s[i:j+1]s[i:j+1]s[i:j+1] 代表的節點在 Trie 中，則可以繼續往下找，並且若該節點是單字的結尾 (isEnd=TrueisEnd = TrueisEnd=True)，則可以進行劃分。

  • 然後檢查 word+s[i]word + s[i]word+s[i] 是否在 Trie 中，由於傳入了 curcurcur 節點，因此檢查 s[i]s[i]s[i] 是否在 cur.childrencur.childrencur.children 中即可。

    • 若 s[i]s[i]s[i] 在 Trie 中，且 nxtnxtnxt 是單字的結尾，則可以進行劃分，將 wordwordword 添加到路徑 pathpathpath 中。此時需要將 curcurcur 重置為 Trie 的根節點，wordwordword 重置為空字串，從頭開始找。

同方法一，由於 C++ 不像 Python 有 join() 方法，因此在 C++ 的範例程式碼中，是直接將所有可能劃分的單字串接起來。在恢復狀態時， Python 是使用 path.pop() 來恢復狀態；而 C++ 的寫法中是直接將 pathpathpath 作為參數傳遞，這樣在遞迴時就不需要恢復狀態。但我個人認為 Python 的寫法更加直觀，可以用 Python 的寫法來理解。

複雜度分析

  • 時間複雜度 O(S+n⋅2n)O(S + n \cdot 2^n)O(S+n⋅2n)，其中 SSS 為 wordDictwordDictwordDict 中所有單字的總長度，nnn 為 sss 的長度。可以視為有 O(2n)O(2^n)O(2n) 個遞迴終點，每個遞迴終點需要 O(n)O(n)O(n) 的時間將劃分的結果串接起來並添加到答案中。
  • 空間複雜度 O(S+n)O(S + n)O(S+n)，不計算答案的空間。空間複雜度主要取決於 Trie 的空間複雜度 O(S)O(S)O(S) 、遞迴調用的深度 O(n)O(n)O(n)、路徑 pathpathpath 的空間複雜度 O(n)O(n)O(n)。

方法三：狀態壓縮，枚舉劃分點

上面兩種方式都是在需要劃分時進行遞迴，那不妨嘗試直接枚舉所有可能的劃分點，這樣就不需要進行遞迴，可以直接得到答案。而長度為 nnn 的字串 sss，可以劃分的劃分點有 n−1n-1n−1 個，因此可以使用狀態壓縮的方式枚舉所有可能的劃分點，即枚舉所有 2n−12^{n-1}2n−1 個子集。
