🔗 🟡 2982. Find Longest Special Substring That Occurs Thrice II 1773

tags: Weekly Contest 378 分組循環 分類討論 排序(Sorting) 堆積(Heap) 計數(Counter) 雜湊表(Hash Table)

這題是 2981 的數據範圍加大版本，暴力解法可以參考該題的 解題紀錄 ，但在本題中使用會 TLE 。

題意

給定一個全由小寫字母組成的字串 sss 。

如果一個字串只包含單一字元，則稱之為 特殊的(Special)* 。例如 "abc""abc""abc" 不是特殊字串，但 "ddd""ddd""ddd"、"zz""zz""zz" 和 "f""f""f" 都是特殊字串。

請返回 sss 中出現至少三次的最長特殊子字串的長度，如果沒有這樣的特殊子字串，則返回 −1-1−1 。

限制

  • 3≤s.length≤5×1053 \leq s.length \leq 5 \times 10^53≤s.length≤5×105

思路：分組循環 + 分類討論

題意中的特殊字串，即為連續的相同字元。因此可以用 分組循環 的思路找出每一段連續的相同字元

方法一：分組循環 + 暴力枚舉 + 計數(Counter)

在取得每一段的長度後，最直接的做法是計算其對不同長度的貢獻。例如一個長度為 kkk 的子串，則其對於長度為 kkk 的特殊子字串的貢獻為 kkk，對於長度為 k−1k-1k−1 的特殊子字串的貢獻為 k−1k-1k−1，以此類推。因此對於每一個字母，可以用一個長度為 n+1n+1n+1 的陣列來記錄每個長度的出現次數，最後再找出出現次數 ≥3\geq 3≥3 的最大長度即可。

需要注意的是，在 Python 中若使用 Counter 來記錄每一個字元的連續長度的貢獻，會導致 TLE，因此需要改用二維陣列來記錄，而最長的特殊子字串長度為
nnn，因此值域為 [0,n][0, n][0,n] 。

此外，還能夠有一些優化：

  • 由於長度為 nnn 的子字串最多只會出現 111 次，長度為 n−1n-1n−1 的子字串最多只會出現 222 次，因此在檢查答案時，可以從 n−2n-2n−2 開始檢查，並且只要找到一個符合條件的即可跳出迴圈。
  • 在計算貢獻時也不用考慮全部的長度，由於長度為 k−2k-2k−2 的特殊子字串的貢獻為 333 ，只需要考慮長度為 kkk、k−1k-1k−1、k−2k-2k−2 的特殊子字串的貢獻即可。

複雜度分析

  • 時間複雜度：O(n+26n)=O(n)\mathcal{O}(n + 26n) = \mathcal{O}(n)O(n+26n)=O(n)
    • 雖然分組循環中有雙重迴圈，但 iii 只會增加 nnn 次；在計算貢獻時，也只會各遍歷一次不重疊的每一段，而每段的總長度也為 nnn，故分組循環的時間複雜度為 O(n)\mathcal{O}(n)O(n)
  • 空間複雜度：O(26n)=O(n)\mathcal{O}(26n) = \mathcal{O}(n)O(26n)=O(n)

方法二：分組循環 + 分類討論 + 排序(Sorting)

如同方法一中的思路，但由於我們只需要考慮出現 333 次以上的情況，因此對於長度為 kkk 的子字串，其實只需要考慮其對於長度為 kkk、k−1k-1k−1、k−2k-2k−2 的特殊子字串的貢獻即可。此外，將每一段的長度由大到小排序後，會發現答案至少為第 333 大的長度，因此只需要考慮前 333 大的長度即可。

同樣是使用分組循環中，將每段的長度加入到該字母的分組 gig_igi​ 中，最後對於每一個字母，找出前 333 大的長度，再 分類討論 即可。若 gig_igi​ 為空，則跳過，否則則將 gig_igi​ 由大到小排序，並假設長度 m≥3m \geq 3m≥3 ，方便以下說明：

  • 由於 gi[0]g_i[0]gi​[0] 可以產生 333 個長度為 gi[0]−2g_i[0]-2gi​[0]−2 的特殊子字串，因此可以更新答案為 max(ans,gi[0]−2)max(ans, g_i[0]-2)max(ans,gi​[0]−2)
  • 由於 gi[0]g_i[0]gi​[0] 可以產生 222 個長度為 gi[0]−1g_i[0]-1gi​[0]−1 的特殊子字串，再加上一個長度為 gi[1]g_i[1]gi​[1] 的特殊子字串，故至少有 333 個長度為 min(gi[0]−1,gi[1])min(g_i[0]-1, g_i[1])min(gi​[0]−1,gi​[1]) 的特殊子字串，因此可以更新答案為 max(ans,min(gi[0]−1,gi[1]))max(ans, min(g_i[0]-1, g_i[1]))max(ans,min(gi​[0]−1,gi​[1]))
  • 由於 gi[0]≥gi[1]≥gi[2]g_i[0] \geq g_i[1] \geq g_i[2]gi​[0]≥gi​[1]≥gi​[2] ，故至少能產生 333 個長度為 gi[2]g_i[2]gi​[2] 的特殊子字串，因此可以更新答案為 max(ans,gi[2])max(ans, g_i[2])max(ans,gi​[2])

初始化答案為 000 ，若答案為 000 則回傳 −1-1−1 。可以透過檢查 mmm 來判斷是否要執行上述的更新操作、或是直接在 gig_igi​ 後加上 222 個 000 ，可以不用考慮 m<3m < 3m<3 的情況，在後續的更新操作中也不會影響答案。

複雜度分析

  • 時間複雜度：O(n+26⋅nlog⁡n)=O(nlog⁡n)\mathcal{O}(n + 26 \cdot n \log n) = \mathcal{O}(n \log n)O(n+26⋅nlogn)=O(nlogn)
    • 分組循環的時間複雜度為 O(n)\mathcal{O}(n)O(n)
    • 分類討論的時間複雜度為 O(26⋅nlog⁡n)=O(nlog⁡n)\mathcal{O}(26 \cdot n \log n) = \mathcal{O}(n \log n)O(26⋅nlogn)=O(nlogn)
  • 空間複雜度：O(n)\mathcal{O}(n)O(n) ，最壞情況下會有 nnn 個長度為 111 的子字串

方法三：分組循環 + 分類討論 + 堆積(Heap)

如同方法二的思路，但既然我們只需要考慮前 333 大的值，那不妨使用 堆積(Heap) 來維護每組的前 333 大的值。在每一段的長度加入到該字母的分組 gig_igi​ 中後，若長度超過 333 個，則將最小的長度移除。在 Python 中可以使用 heapq 模組中的 heappush 和 heappop 來維護 Min Heap，而在 C++ 中可以使用 priority_queue 來維護 Min Heap。

新建一個長度為 333 的陣列 tmptmptmp 來存放 gig_igi​ 的前 333 大的值，並從將堆頂的元素依次放到 tmp[m−1],tmp[m−2],tmp[m−3]tmp[m-1], tmp[m-2], tmp[m-3]tmp[m−1],tmp[m−2],tmp[m−3] 中，其中 mmm 為 gig_igi​ 的長度，這種寫法可以考慮到 m<3m < 3m<3 的情況，讀者可以自行驗證。如此 tmptmptmp 中的元素即為 gig_igi​ 的前 333 大的值，最後如同方法二中的 分類討論 思路，根據 tmptmptmp 的值來更新答案。

複雜度分析

  • 時間複雜度：O(nlog⁡3+26⋅3log⁡3)=O(n)\mathcal{O}(n \log 3 + 26 \cdot 3 \log 3) = \mathcal{O}(n)O(nlog3+26⋅3log3)=O(n)
    • 分組循環的時間複雜度為 O(n+nlog⁡3)=O(n)\mathcal{O}(n + n \log 3) = \mathcal{O}(n)O(n+nlog3)=O(n) ，因為 heappush 和 heappop 的時間複雜度為 O(log⁡3)\mathcal{O}(\log 3)O(log3)，且最多只會執行 O(n)O(n)O(n) 次
    • 分類討論的時間複雜度為 O(26⋅3log⁡3)=O(1)\mathcal{O}(26 \cdot 3 \log 3) = \mathcal{O}(1)O(26⋅3log3)=O(1) 。
  • 空間複雜度：O(26⋅3)=O(1)\mathcal{O}(26 \cdot 3) = \mathcal{O}(1)O(26⋅3)=O(1)
