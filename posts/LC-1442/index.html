🔗 🟡 1442. Count Triplets That Can Form Two Arrays of Equal XOR 1525

tags: Weekly Contest 188 前綴和(Prefix Sum) 雜湊表(Hash Table) 位運算(Bit Manipulation)

題意

給定一個整數陣列 arrarrarr。

從陣列中選擇三個索引 iii、jjj 和 kkk，其中 (0≤i<j≤k<arr.length)(0 \leq i < j \leq k < \text{arr.length})(0≤i<j≤k<arr.length)。

定義 aaa 和 bbb 如下：

  • a=arr[i]⊕arr[i+1]⊕…⊕arr[j−1]a = arr[i] \oplus arr[i + 1] \oplus \ldots \oplus arr[j - 1]a=arr[i]⊕arr[i+1]⊕…⊕arr[j−1]
  • b=arr[j]⊕arr[j+1]⊕…⊕arr[k]b = arr[j] \oplus arr[j + 1] \oplus \ldots \oplus arr[k]b=arr[j]⊕arr[j+1]⊕…⊕arr[k]

返回滿足 a=ba = ba=b 的三元組 (i,j,k)(i, j, k)(i,j,k) 的數量。

限制

  • 1≤arr.length≤3001 \leq arr.length \leq 3001≤arr.length≤300
  • 1≤arr[i]≤1081 \leq arr[i] \leq 10^81≤arr[i]≤108

思路：前綴和(Prefix Sum)

首先，可以注意到 aaa 和 bbb 都是子陣列的 XOR 和，所以可以使用 前綴和(Prefix Sum) 的方式來預處理 arrarrarr ，使得可以在 O(1)O(1)O(1) 的時間內計算 aaa 和 bbb。

令 pre[i]pre[i]pre[i] 表示 arr[0]⊕arr[1]⊕…⊕arr[i−1]arr[0] \oplus arr[1] \oplus \ldots \oplus arr[i - 1]arr[0]⊕arr[1]⊕…⊕arr[i−1] ，則

  • a=pre[j]⊕pre[i]a = pre[j] \oplus pre[i]a=pre[j]⊕pre[i]
  • b=pre[k+1]⊕pre[j]b = pre[k + 1] \oplus pre[j]b=pre[k+1]⊕pre[j]

方法一：三重迴圈

根據題意，枚舉三個下標 iii、jjj 和 kkk，其中 i<j≤ki < j \leq ki<j≤k，並檢查 a=ba = ba=b 是否成立，若成立則累加答案即可。

由於 n≤300n \leq 300n≤300，所以 O(n3)O(n^3)O(n3) 的時間複雜度是可以通過的。

而在檢查 pre[i]⊕pre[j]=pre[j]⊕pre[k+1]pre[i] \oplus pre[j] = pre[j] \oplus pre[k + 1]pre[i]⊕pre[j]=pre[j]⊕pre[k+1] 時，可以發現等號兩邊都有 pre[j]pre[j]pre[j]，因此在做判斷的時候，可以直接比較 pre[i]pre[i]pre[i] 和 pre[k+1]pre[k + 1]pre[k+1] 是否相等即可。這個性質可以幫助我們優化時間複雜度，會在之後的方法中提到。

複雜度分析

  • 時間複雜度 O(n3)O(n^3)O(n3)
  • 空間複雜度 O(n)O(n)O(n)

方法二：二重迴圈

從方法一中，我們注意到了在判斷 a=ba = ba=b 的時候，與 jjj 有關的部分可以省略，因此可以將三重迴圈的 jjj 部分省略，只保留兩重迴圈。

再來需要考慮的是當 pre[i]==pre[k+1]pre[i] == pre[k + 1]pre[i]==pre[k+1] 時，有多少組 (i,j,k)(i, j, k)(i,j,k) 滿足條件。這裡可以從方法一中思考，此時滿足條件的 (i,j,k)(i, j, k)(i,j,k) 為 (i,i+1,k),(i,i+2,k),...,(i,k,k)(i, i+1, k), (i, i+2, k), ..., (i, k, k)(i,i+1,k),(i,i+2,k),...,(i,k,k)，總共有 k−ik - ik−i 種組合，因此可以直接計算 k−ik - ik−i 並累加到答案中。

複雜度分析

  • 時間複雜度 O(n2)O(n^2)O(n2)
  • 空間複雜度 O(n)O(n)O(n)

方法三：雜湊表(Hash Table)

由於能夠滿足條件的 (i,j,k)(i, j, k)(i,j,k) 皆滿足 pre[i]==pre[k+1]pre[i] == pre[k + 1]pre[i]==pre[k+1] ，對於每個 kkk ，我們只在乎能夠使得 pre[i]==pre[k+1]pre[i] == pre[k + 1]pre[i]==pre[k+1] 的 iii 有哪些，因此我們可以建立一個雜湊表 pospospos ，保存每個 pre[i]pre[i]pre[i] 出現的位置。

接著使用一重迴圈枚舉 kkk，對於每個 kkk，檢查 pre[k+1]pre[k + 1]pre[k+1] 是否在 pospospos 中，若在則 pos[pre[k+1]]pos[pre[k + 1]]pos[pre[k+1]] 中的每個 iii ，其對答案的貢獻為 k−ik - ik−i ，累加到答案中，這部分和方法二中的思路相同。最後再把當前的 kkk 加入 pos[pre[k]]pos[pre[k]]pos[pre[k]] 中，以便後續的計算。

複雜度分析

  • 時間複雜度 O(n2)O(n^2)O(n2)
  • 空間複雜度 O(n)O(n)O(n)

方法三優化：雜湊表(Hash Table) + 一重迴圈

方法三中，雖然已經節省了枚舉 i,ji, ji,j 的時間，但是在枚舉 kkk 的時候，仍然需要遍歷 pos[pre[k+1]]pos[pre[k + 1]]pos[pre[k+1]] 中的每個 iii，在最壞情況下，時間複雜度仍然是 O(n2)O(n^2)O(n2) 。

而是否存在 O(n)O(n)O(n) 的解法呢？不妨從貢獻的角度來思考，對於 pos[pre[k+1]]pos[pre[k+1]]pos[pre[k+1]] 中的每個 i，其對答案的貢獻為 k−ik - ik−i，可以將其拆分為兩部分：

  • kkk 對答案的總貢獻為 k×mk \times mk×m，其中 mmm 為 pos[pre[k+1]]pos[pre[k+1]]pos[pre[k+1]] 的長度，即下標的個數。
  • iii 對答案的總貢獻為 ∑i∈pos[pre[k+1]]i\sum_{i \in pos[pre[k+1]]} i∑i∈pos[pre[k+1]]​i。

因此，可以將 pos[pre[k+1]]pos[pre[k+1]]pos[pre[k+1]] 的長度和下標的總和都保存下來，這樣在枚舉 kkk 的時候，只需要計算 k×mk \times mk×m 和 ∑i∈pos[pre[k+1]]i\sum_{i \in pos[pre[k+1]]} i∑i∈pos[pre[k+1]]​i 即可。

這裡使用兩個 雜湊表(Hash Table) cntcntcnt 和 tottottot 來保存 pos[pre[k+1]]pos[pre[k+1]]pos[pre[k+1]] 的長度以及總和，這樣就可以在一重迴圈中完成計算。

複雜度分析

  • 時間複雜度 O(n)O(n)O(n)
  • 空間複雜度 O(n)O(n)O(n)
