🔗 🔴 719. Find K-th Smallest Pair Distance

tags: Weekly Contest 56 陣列(Array) 排序(Sorting) 二分搜尋(Binary Search) 雙指針(Two Pointers) 滑動窗口(Sliding Window)

題意

一對整數 (a,b)(a, b)(a,b) 由整數 aaa 和 bbb 組成，其距離定義為 aaa 和 bbb 之間的絕對差 ∣a−b∣|a - b|∣a−b∣。

給定一個整數陣列 numsnumsnums 和一個整數 kkk，返回所有 nums[i]nums[i]nums[i] 和 nums[j]nums[j]nums[j] 配對中第 kkk 小的距離，其中 0≤i<j<nums.length0 \leq i < j < nums.length0≤i<j<nums.length。

約束條件

  • n=nums.lengthn = nums.lengthn=nums.length
  • 2≤n≤1042 \leq n \leq 10^42≤n≤104
  • 0≤nums[i]≤1060 \leq nums[i] \leq 10^60≤nums[i]≤106
  • 1≤k≤n∗(n−1)21 \leq k \leq \frac{n * (n - 1)}{2}1≤k≤2n∗(n−1)​

思路：二分搜尋(Binary Search)

為方便敘述，以下標配對 (i,j)(i, j)(i,j) 表示 nums[i]nums[i]nums[i] 和 nums[j]nums[j]nums[j] 的配對，即 (nums[i],nums[j])(nums[i], nums[j])(nums[i],nums[j]) 。

雖然題目中的數對 (i,j)(i, j)(i,j) 需要滿足 i<ji < ji<j，但注意到距離的定義是 絕對值 ，因此從原陣列中找數對 (i,j)(i, j)(i,j) ，等同於在 排序 陣列中找數對 (i,j)(i, j)(i,j)。

最暴力的做法是根據所有可能的 (i,j)(i, j)(i,j) 的配對，生成所有距離，然後排序後取出第 kkk 小的距離即可。但所有可能的配對有 n∗(n−1)2\frac{n * (n - 1)}{2}2n∗(n−1)​ 個，故暴力法的時間複雜度為 O(n2log⁡n2)O(n^2 \log n^2)O(n2logn2) 。而在約束條件中 nnn 為 10410^4104 ，顯然是會超時的。

如果我們將所有數對的距離進行 計數(Counting) ，那我們可以得到一個距離和數量的陣列 cntcntcnt，其中 cnt[i]cnt[i]cnt[i] 表示所有 (i,j)(i, j)(i,j) 數對中，距離為 iii 的數對數量。接著對 cntcntcnt 做 前綴和(Prefix Sum) 得到 sss，則 s[k]s[k]s[k] 表示 距離 ≤k\leq k≤k 的數對數量。由於 cnt[i]cnt[i]cnt[i] 皆為非負整數，因此 sss 具有 遞增 性質。

而若 s[k]=y,s[k−1]=xs[k] = y, s[k-1] = xs[k]=y,s[k−1]=x，則代表第 x+1,x+2,...,yx+1, x+2, ..., yx+1,x+2,...,y 小的距離皆為 kkk ，這裡可以代範例下去理解。

因此若可以在不生成所有數對的情況下，就可以確定 s[k]s[k]s[k] 的值，就可以對所有可能的距離做 二分搜尋(Binary Search) 。又數對的距離最小為 000 ，最大為 U−LU - LU−L，其中 UUU 和 LLL 分別為整數陣列的最大值和最小值，因此二分的範圍為 [0,U−L][0, U-L][0,U−L] 。

令 check(k) 表示 s[k]s[k]s[k] ，即距離 ≤k\leq k≤k 的數對數量，則有兩種方式可以直接確定 s[k]s[k]s[k] 的值：

  1. 二分搜尋(Binary Search)
    • 枚舉所有可能的右端點 jjj，使用二分搜尋找出左端點 iii，使得數對 (i,j),(i+1,j),...,(j−1,j)(i, j), (i+1, j), ..., (j-1, j)(i,j),(i+1,j),...,(j−1,j) 的距離都小於等於 kkk 。
  2. 滑動窗口(Sliding Window)
    • 同樣枚舉所有可能的右端點 jjj，但不用重新計算，而是逐漸縮小左端點 iii，使得數對 (i,j),(i+1,j),...,(j−1,j)(i, j), (i+1, j), ..., (j-1, j)(i,j),(i+1,j),...,(j−1,j) 的距離都小於等於 kkk 。

方法一：二分搜尋(Binary Search)

在確定右端點 jjj 之後，若要使得數對 (i,j),(i+1,j),...,(j−1,j)(i, j), (i+1, j), ..., (j-1, j)(i,j),(i+1,j),...,(j−1,j) 這 j−ij - ij−i 個數對的距離都小於等於 kkk ，則 nums[i]nums[i]nums[i] 的值需要至少為 k−nums[j]k - nums[j]k−nums[j] ，因此可以使用 bisect_left 或 lower_bound 來找出左端點 iii ，即第一個滿足 nums[i]≥k−nums[j]nums[i] \geq k - nums[j]nums[i]≥k−nums[j] 的 iii 。

枚舉所有可能的 jjj，累加所有滿足條件的數對數量即可。

複雜度分析

  • 時間複雜度：O(nlog⁡n×log⁡(U−L))\mathcal{O}(n \log n \times \log (U - L))O(nlogn×log(U−L))，其中 nnn 為陣列的長度，UUU 和 LLL 分別為整數陣列的最大值和最小值。
    • 排序陣列需要 O(nlog⁡n)O(n \log n)O(nlogn) 的時間。
    • 對答案做二分搜尋需要 O(log⁡(U−L))O(\log (U - L))O(log(U−L)) 的時間。
    • 計算 check(k) 需要 O(nlog⁡n)O(n \log n)O(nlogn) 的時間。
  • 空間複雜度：O(log⁡n)\mathcal{O}(\log n)O(logn)，即排序陣列的空間複雜度。

方法二：滑動窗口(Sliding Window)

由於陣列已經排序過，因此 (i,j)(i, j)(i,j) 的距離會比 (i,j−1)(i, j - 1)(i,j−1) 大，故若 (i,j)(i, j)(i,j) 的不滿足條件(即距離大於 kkk)，則(i - 1, j) 的距離只會更大，顯然也不滿足條件。因此我們實際上不需要重新計算左端點 iii，只需要檢查先前的左端點 iii 和當前的右端點 jjj 是否符合條件即可。若不滿足條件，再移動左端點 iii 直到滿足條件即可。

雖然看似是 O(n2)O(n^2)O(n2) 的時間複雜度，但實際上只需要 O(n)O(n)O(n) 的時間複雜度，因為左端點 iii 最多只會從 000 走到 n−1n-1n−1 。

複雜度分析

  • 時間複雜度：O(nlog⁡n+nlog⁡(U−L))\mathcal{O}(n \log n + n \log (U - L))O(nlogn+nlog(U−L))，其中 nnn 為陣列的長度，UUU 和 LLL 分別為整數陣列的最大值和最小值。
    • 排序陣列需要 O(nlog⁡n)O(n \log n)O(nlogn) 的時間。
    • 對答案做二分搜尋需要 O(log⁡(U−L))O(\log (U - L))O(log(U−L)) 的時間。
    • 計算 check(k) 需要 O(n)O(n)O(n) 的時間。
  • 空間複雜度：O(log⁡n)\mathcal{O}(\log n)O(logn)，即排序陣列的空間複雜度。
